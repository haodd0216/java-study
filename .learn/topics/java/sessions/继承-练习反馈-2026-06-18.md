编译和运行均通过。

## 完成情况

已完成：

- `Shape` 作为父类，`name` 私有，只提供 getter，不提供 setter。
- `Shape` 提供默认 `area()` 返回 0，以及 `describe()` 方法。
- `Circle extends Shape`，通过 `super("圆形")` 设置名称。
- `Rectangle extends Shape`，通过 `super("矩形")` 设置名称。
- `Circle` 和 `Rectangle` 都用 `@Override` 重写 `area()`。
- `Rectangle` 重写 `describe()`，并用 `super.describe()` 复用父类逻辑。
- `describe()` 调用 `area()` 时体现了子类的实际计算结果。

## 做得好的地方

1. `super("圆形")` 和 `super("矩形")` 用法正确，父类构造在第一行。
2. 三个类都正确使用 `@Override`，方法重写没有写错方法名。
3. `Rectangle.describe()` 里先 `super.describe()` 再补充打印，很好地体现了“复用父类 + 扩展子类”。
4. `Circle.area()` 使用 `Math.PI * Math.pow(radius, 2)`，写法清晰。
5. 父类 `name` 私有，子类通过继承的 `describe()` 间接使用，没有破坏封装。

## 需要注意的地方

1. 题目要求 `radius`、`width`、`height` 必须大于 0，也就是应拒绝 `<= 0`。当前判断是 `radius < 0` 和 `width < 0 || height < 0`，没有拒绝 0。如果传入 0，会创建出面积为 0 的图形。建议改成：

```java
if (radius <= 0) {
    System.out.println("半径必须大于0");
    return;
}
```

2. 提示文字和题目要求略有差异（题目是 `半径必须大于0`、`宽和高必须大于0`），含义正确，如需严格对齐可调整。

3. 一个更深层的点（了解即可）：当前构造方法在非法输入时只是打印并 `return`，对象仍会被创建，只是字段是默认值 0。这和你在封装练习中遇到的情况一样，等学到异常后可以用 `throw` 更严格地阻止非法对象。

## 学习判断

本次练习把继承的核心机制都用到了：

- `extends` 表达 is-a；
- `super(...)` 初始化父类部分；
- `@Override` 重写父类方法；
- `super.方法()` 复用父类逻辑；
- 继承与封装配合。

继承这一节可以认为达到当前阶段的较好掌握水平。下一步适合进入多态，因为多态正是建立在“父类引用 + 方法重写”的基础上。
