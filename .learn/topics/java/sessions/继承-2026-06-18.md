# 继承

日期：2026-06-18
主题：Java / 面向对象编程 / 继承

## 1. 定位：继承解决什么问题？

你已经学过类、对象和封装。

封装关注的是：

> 一个类内部的数据应该如何被保护，以及外部应该通过什么方法使用它。

继承关注的是另一个问题：

> 多个类有共同的属性和行为时，如何把公共部分抽出来复用？

例如你要写三个类：

- `Student` 学生；
- `Teacher` 老师；
- `Manager` 管理员。

它们可能都有：

- 姓名 `name`；
- 年龄 `age`；
- `eat()` 方法；
- `sleep()` 方法。

如果每个类都重复写一遍，就会有很多重复代码。

继承的思想是：

> 把共同的部分放到父类里，子类继承父类，再添加自己特有的内容。

例如：

- 父类：`Person`
- 子类：`Student extends Person`
- 子类：`Teacher extends Person`

这样 `Student` 和 `Teacher` 都可以复用 `Person` 中的字段和方法。

## 2. 类比：分类和继承

可以把继承理解成“分类”。

比如：

```text
动物
├── 狗
├── 猫
└── 鸟
```

所有动物都有一些共同点：

- 有名字；
- 会吃东西；
- 会移动。

狗有自己的特点：

- 会看家；
- 会汪汪叫。

猫有自己的特点：

- 会抓老鼠；
- 会喵喵叫。

所以可以说：

> 狗是一种动物，猫也是一种动物。

在 Java 里，这种“是一种”的关系就适合用继承表达。

```java
class Animal {
}

class Dog extends Animal {
}

class Cat extends Animal {
}
```

这里 `Dog extends Animal` 的意思是：

> Dog 是 Animal 的一种。

## 3. 核心语法：extends

Java 使用 `extends` 表示继承。

```java
class 父类 {
    // 公共字段和方法
}

class 子类 extends 父类 {
    // 子类特有字段和方法
}
```

例如：

```java
class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }

    public void eat() {
        System.out.println(name + " 正在吃饭");
    }
}

class Student extends Person {
    private String studentId;

    public Student(String name, int age, String studentId) {
        super(name, age);
        this.studentId = studentId;
    }

    public String getStudentId() {
        return studentId;
    }

    public void study() {
        System.out.println(getName() + " 正在学习");
    }
}
```

使用：

```java
public class Main {
    public static void main(String[] args) {
        Student s = new Student("小明", 18, "S001");

        System.out.println(s.getName());
        System.out.println(s.getAge());
        System.out.println(s.getStudentId());

        s.eat();
        s.study();
    }
}
```

这里：

- `Student` 继承了 `Person`；
- `getName()`、`getAge()`、`eat()` 来自父类；
- `studentId`、`getStudentId()`、`study()` 是学生自己的内容。

## 4. 父类和子类的关系

继承中的两个角色：

```text
父类 / 超类 / 基类：被继承的类
子类 / 派生类：继承别人的类
```

例如：

```java
class Person {
}

class Student extends Person {
}
```

其中：

- `Person` 是父类；
- `Student` 是子类；
- `Student` 拥有 `Person` 中可以继承的成员；
- `Student` 还可以添加自己的成员。

可以理解成：

```text
子类 = 父类的公共能力 + 子类自己的能力
```

但注意：子类不是简单复制了一份父类代码，而是通过继承关系获得了使用父类成员的能力。

## 5. private 字段不会被子类直接访问

你已经学过封装，所以这里要特别注意：

> 父类里的 `private` 字段，子类不能直接访问。

例如：

```java
class Person {
    private String name;
}

class Student extends Person {
    public void printName() {
        System.out.println(name); // 编译错误
    }
}
```

虽然 `Student` 继承了 `Person`，但 `name` 是 `private`，只能在 `Person` 类内部直接访问。

如果子类要使用父类数据，应该通过父类提供的方法：

```java
class Person {
    private String name;

    public String getName() {
        return name;
    }
}

class Student extends Person {
    public void printName() {
        System.out.println(getName());
    }
}
```

这说明：

> 继承不会破坏封装。

子类虽然继承父类，但仍然要尊重父类的访问控制。

## 6. super：调用父类构造方法

子类对象创建时，父类部分也必须先创建好。

所以子类构造方法里通常要调用父类构造方法。

使用：

```java
super(参数列表);
```

例如：

```java
class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

class Student extends Person {
    private String studentId;

    public Student(String name, int age, String studentId) {
        super(name, age);
        this.studentId = studentId;
    }
}
```

这里：

```java
super(name, age);
```

表示调用父类 `Person` 的构造方法，先初始化父类中的 `name` 和 `age`。

注意：

> `super(...)` 必须写在子类构造方法的第一行。

错误写法：

```java
public Student(String name, int age, String studentId) {
    this.studentId = studentId;
    super(name, age); // 编译错误
}
```

正确写法：

```java
public Student(String name, int age, String studentId) {
    super(name, age);
    this.studentId = studentId;
}
```

## 7. 如果父类没有无参构造方法

如果父类写了有参构造方法，而且没有无参构造方法：

```java
class Person {
    public Person(String name, int age) {
    }
}
```

那么子类构造方法必须显式调用：

```java
super(name, age);
```

否则会报错。

为什么？

因为 Java 创建子类对象时，一定要先创建父类部分。

如果你不写 `super(...)`，Java 默认会尝试调用：

```java
super();
```

也就是父类无参构造方法。

但父类没有无参构造方法时，就会失败。

所以：

```java
class Student extends Person {
    public Student(String studentId) {
        this.studentId = studentId; // 还没调用父类构造，会出问题
    }
}
```

应该写成：

```java
class Student extends Person {
    private String studentId;

    public Student(String name, int age, String studentId) {
        super(name, age);
        this.studentId = studentId;
    }
}
```

## 8. 子类可以添加自己的字段和方法

继承不是只能复用，也可以扩展。

例如：

```java
class Teacher extends Person {
    private String subject;

    public Teacher(String name, int age, String subject) {
        super(name, age);
        this.subject = subject;
    }

    public String getSubject() {
        return subject;
    }

    public void teach() {
        System.out.println(getName() + " 正在讲授 " + subject);
    }
}
```

`Teacher` 继承了 `Person` 的：

- `name`；
- `age`；
- `getName()`；
- `getAge()`；
- `eat()`。

同时自己新增：

- `subject`；
- `getSubject()`；
- `teach()`。

这就是：

> 继承 = 复用父类 + 扩展子类。

## 9. 方法重写：子类改变父类行为

有时子类继承了父类的方法，但父类的方法不够具体。

例如：

```java
class Animal {
    public void makeSound() {
        System.out.println("动物发出声音");
    }
}

class Dog extends Animal {
    public void makeSound() {
        System.out.println("狗汪汪叫");
    }
}
```

`Dog` 里重新写了 `makeSound()`，这叫方法重写。

建议加上 `@Override`：

```java
class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("狗汪汪叫");
    }
}
```

`@Override` 的作用是告诉编译器：

> 我这里是想重写父类方法，请帮我检查有没有写错。

如果方法名、参数写错，编译器会报错。

例如：

```java
class Dog extends Animal {
    @Override
    public void makesound() {
        System.out.println("狗汪汪叫");
    }
}
```

因为大小写不一致，`makesound` 并没有重写 `makeSound`，编译器会提醒你。

## 10. super：调用父类方法

除了调用父类构造方法，`super` 也可以调用父类方法。

例如：

```java
class Animal {
    public void makeSound() {
        System.out.println("动物发出声音");
    }
}

class Dog extends Animal {
    @Override
    public void makeSound() {
        super.makeSound();
        System.out.println("狗汪汪叫");
    }
}
```

调用：

```java
Dog dog = new Dog();
dog.makeSound();
```

输出：

```text
动物发出声音
狗汪汪叫
```

这里：

```java
super.makeSound();
```

表示调用父类版本的 `makeSound()`。

## 11. Java 只支持单继承

Java 中，一个类只能直接继承一个父类。

可以：

```java
class Student extends Person {
}
```

不可以：

```java
class Student extends Person, Learner {
}
```

也就是说：

> Java 类只支持单继承。

但一个类可以间接形成多层继承：

```java
class Animal {
}

class Mammal extends Animal {
}

class Dog extends Mammal {
}
```

这里：

```text
Animal -> Mammal -> Dog
```

`Dog` 间接也是一种 `Animal`。

以后学接口时，会看到 Java 通过接口解决“多个能力组合”的问题。

## 12. 所有类都间接继承 Object

在 Java 中，如果一个类没有写 `extends`，它默认继承 `Object`。

例如：

```java
class Person {
}
```

可以理解为：

```java
class Person extends Object {
}
```

`Object` 是所有类的顶层父类。

所以每个对象都有一些来自 `Object` 的方法，比如：

- `toString()`；
- `equals()`；
- `hashCode()`。

这些以后会在常用核心 API 里详细学。

## 13. 什么时候适合用继承？

继承适合表达“is-a”关系，也就是“子类是父类的一种”。

适合：

```text
Student 是 Person 的一种
Teacher 是 Person 的一种
Dog 是 Animal 的一种
Cat 是 Animal 的一种
```

不适合：

```text
Car 有 Engine
Computer 有 Keyboard
Order 有 Product
```

这些是“has-a”关系，更适合用字段组合，而不是继承。

例如，不应该写：

```java
class Car extends Engine {
}
```

因为汽车不是发动机的一种，汽车只是有一个发动机。

更合理：

```java
class Car {
    private Engine engine;
}
```

所以判断继承是否合适，可以问：

> A 是 B 的一种吗？

如果是，可能适合继承。

如果只是 A 拥有 B，就更适合组合。

## 14. 继承和封装的关系

你刚学过封装，所以要把这两个概念连起来。

继承不是为了让子类随便操作父类内部数据。

父类仍然应该封装自己的字段：

```java
class Person {
    private String name;

    public String getName() {
        return name;
    }
}
```

子类通过父类提供的方法使用数据：

```java
class Student extends Person {
    public void study() {
        System.out.println(getName() + " 正在学习");
    }
}
```

所以：

```text
封装负责保护内部状态
继承负责复用和扩展能力
```

好的继承设计，应该尊重父类封装。

## 15. 一个完整例子

```java
class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }

    public void introduce() {
        System.out.println("大家好，我叫" + name + "，今年" + age + "岁");
    }
}

class Student extends Person {
    private String studentId;

    public Student(String name, int age, String studentId) {
        super(name, age);
        this.studentId = studentId;
    }

    public String getStudentId() {
        return studentId;
    }

    public void study() {
        System.out.println(getName() + " 正在学习");
    }

    @Override
    public void introduce() {
        super.introduce();
        System.out.println("我的学号是" + studentId);
    }
}

public class Main {
    public static void main(String[] args) {
        Student s = new Student("小明", 18, "S001");

        s.introduce();
        s.study();
    }
}
```

输出：

```text
大家好，我叫小明，今年18岁
我的学号是S001
小明 正在学习
```

这个例子里：

- `Person` 保存公共字段和公共方法；
- `Student` 继承 `Person`；
- `Student` 新增 `studentId` 和 `study()`；
- `Student` 重写 `introduce()`；
- 子类通过 `super(...)` 初始化父类部分；
- 子类通过 `super.introduce()` 复用父类方法。

## 16. 常见误区

### 误区一：继承就是为了少写代码

少写重复代码只是结果之一，不是唯一目的。

继承更重要的是表达“is-a”关系。

如果只是为了复用代码，但两者不是同一类事物，强行继承会让设计变奇怪。

### 误区二：子类可以直接访问父类所有字段

不对。

父类 `private` 字段，子类不能直接访问。

子类应该通过父类提供的 public 或 protected 方法访问。

### 误区三：Java 可以继承多个类

不对。

Java 一个类只能直接继承一个父类。

### 误区四：父类构造方法会被子类继承

不对。

构造方法不会被继承。

子类只能通过 `super(...)` 调用父类构造方法。

### 误区五：所有相似的类都应该用继承

不对。

相似不等于继承。

只有当子类确实是父类的一种时，继承才更合理。

## 17. 判断是否该用继承

可以按这个顺序判断：

1. A 是 B 的一种吗？
2. B 中是否有 A 可以复用的公共属性和行为？
3. A 是否需要在 B 的基础上扩展自己的字段或方法？
4. 子类是否能遵守父类的规则？
5. 如果只是“拥有某个对象”，是否更适合用字段组合？

如果第 1 点不成立，就不要急着用继承。
