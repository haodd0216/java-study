# 内部类与匿名类

日期：2026-07-03
主题：Java / 面向对象编程 / 内部类与匿名类

## 1. 定位：内部类解决什么问题？

你已经掌握了抽象类和接口。以你的通知系统为例：

```java
// Trackable 的三个实现：EmailNotification、SMSNotification、PushNotification
// 每个都是一个独立的 .java 文件或独立类
```

但有时候，你只需要一个**临时用一次**的类。比如：

- 给按钮绑定一个点击事件，只需要一个对象，不需要专门建一个类文件
- 某个类的实现逻辑很简单，只在一个地方用，单独建文件小题大做
- 一个类内部有复杂的子结构，在外部单独定义会失去访问上下文

内部类就是为了这些场景：

> 在一个类的内部定义另一个类，让这个类的生命周期和作用范围**局限在需要它的地方**。

---

## 2. 类比：工具箱里的专用工具

```
一个大工具箱（外部类）：
├── 常用的螺丝刀、扳手挂在外侧（外部类的字段和方法）
└── 工具箱暗格里藏着一个小工具（内部类）
    —— 这个小工具只有你知道，专用于某个特殊螺丝
    —— 你能随时拿到它，它也能直接摸到工具箱里其他工具
```

**关键特性**：内部类可以访问外部类的私有成员，就像它属于这个"家庭"一样。

---

## 3. 四种内部类

Java 有四种内部类，按定义位置和作用域区分：

| 类型 | 定义位置 | 关键字 | 典型用途 |
|------|----------|--------|----------|
| 成员内部类 | 类内部、方法外部 | `class` | 紧密关联的辅助类 |
| 静态嵌套类 | 类内部，`static` | `static class` | 与外部类弱关联的工具类 |
| 局部内部类 | 方法内部 | `class` | 方法内的临时类 |
| 匿名内部类 | 表达式里 | `new 接口/类() { }` | 一次性实现（最常见） |

### 3.1 成员内部类

定义在类的内部、方法的外部。和字段、方法平级。

```java
class University {
    private String name = "清华大学";

    // 成员内部类
    class Department {
        private String deptName;

        public Department(String deptName) {
            this.deptName = deptName;
        }

        public void info() {
            // 内部类可以直接访问外部类的私有成员！
            System.out.println(name + " - " + deptName);
        }
    }

    public void createDepartment() {
        Department dept = new Department("计算机系");
        dept.info();  // 清华大学 - 计算机系
    }
}
```

**创建内部类对象**（在外部）：

```java
University uni = new University();
University.Department dept = uni.new Department("数学系");
dept.info();  // 清华大学 - 数学系
```

语法：`外部类对象.new 内部类()`。因为成员内部类必须依附于一个外部类实例。

### 3.2 静态嵌套类

加了 `static` 的内部类。不依赖外部类实例。

```java
class MathUtils {
    // 静态嵌套类
    public static class Calculator {
        public int add(int a, int b) {
            return a + b;
        }
    }
}

// 使用：不需要外部类实例
MathUtils.Calculator calc = new MathUtils.Calculator();
calc.add(1, 2);  // 3
```

和普通外部类的唯一区别：它可以访问外部类的 `static` 成员（包括 private 的）。

### 3.3 局部内部类

定义在**方法内部**的类，作用域仅限于该方法。

```java
public void doSomething() {
    // 局部内部类
    class TemporaryHelper {
        public void help() {
            System.out.println("临时帮忙");
        }
    }

    TemporaryHelper helper = new TemporaryHelper();
    helper.help();
}
```

很少用——它的功能几乎被匿名内部类覆盖了。

### 3.4 匿名内部类 ⭐（最重要）

**没有名字的类，在创建对象的同时定义类体**。

这是你最需要掌握的，因为它和你刚学的接口、抽象类直接相关。

```java
// 不用匿名内部类（传统方式）：
//   1. 定义一个新类 MyTracker implements Trackable
//   2. 实现方法
//   3. new MyTracker()

// 用匿名内部类（一步搞定）：
Trackable tracker = new Trackable() {
    @Override
    public String getChannelName() {
        return "临时渠道";
    }

    @Override
    public String getDeliveryStatus() {
        return "已送达";
    }
};

// 直接用！
System.out.println(tracker.getChannelName());  // 临时渠道
```

**语法解读**：

```
new Trackable() { ... }
│   │            │
│   └─ 接口名    └─ 类体（实现了接口的方法）
└─ new = 创建对象
```

> `new Trackable() { ... }` 的意思是：**创建一个实现了 Trackable 接口的匿名类的对象**。不是 `new` 接口（接口不能 new），而是 new 了一个"没有名字的、实现了 Trackable 的类"。

## 4. 匿名内部类 + 多态 = 威力组合

你已经知道：接口类型变量可以引用实现类对象（接口多态）。

匿名内部类让这个能力更进一步——**不需要单独写一个实现类**：

```java
// 你的通知系统里的 Trackable 数组
Trackable[] trackables = new Trackable[3];

// 用匿名内部类，不需要 EmailNotification 实现 Trackable
trackables[0] = new Trackable() {
    @Override
    public String getChannelName() { return "邮件"; }

    @Override
    public String getDeliveryStatus() { return "邮件已送达"; }
};

trackables[1] = new Trackable() {
    @Override
    public String getChannelName() { return "短信"; }

    @Override
    public String getDeliveryStatus() { return "短信已送达"; }
};

// 直接遍历 — 接口多态！
for (Trackable t : trackables) {
    System.out.println("[" + t.getChannelName() + "] " + t.getDeliveryStatus());
}
```

这和你的 NotificationSystem 里做的是一回事，但**省掉了 EmailNotification 等类对 Trackable 的实现**——匿名类让一次性实现变得极其简洁。

### 匿名类也可以继承抽象类

```java
// 不用定义 Circle 类，直接创建匿名 Shape 子类
Shape anonymousCircle = new Shape("匿名圆形") {
    @Override
    public double area() {
        return Math.PI * 3 * 3;  // 半径固定为 3
    }
};

System.out.println(anonymousCircle.area());  // 28.27...
```

### Lambda 表达式：匿名类的简化版

如果接口只有一个抽象方法（叫**函数式接口**），匿名类可以进一步简化成 Lambda：

```java
// 匿名内部类写法
Trackable t = new Trackable() {
    @Override
    public String getChannelName() { return "Channel"; }
    @Override  // ← Trackable 有两个抽象方法，不能用 Lambda
    public String getDeliveryStatus() { return "OK"; }
};

// 如果接口只有一个方法，比如：
interface Runnable {
    void run();
}

// 匿名内部类写法
Runnable r1 = new Runnable() {
    @Override
    public void run() { System.out.println("running"); }
};

// Lambda 写法（更简洁）
Runnable r2 = () -> System.out.println("running");
```

> Lambda 只能用于**只有一个抽象方法**的接口。如果有多个抽象方法，还是需要用匿名内部类。

## 5. 完整例子：把匿名内部类融入你的知识体系

```java
public class InnerClassDemo {
    public static void main(String[] args) {
        // 1. 匿名内部类实现接口
        //    这就是你的 Trackable 接口多态
        Trackable anonymousTracker = new Trackable() {
            @Override
            public String getChannelName() { return "匿名渠道"; }

            @Override
            public String getDeliveryStatus() { return "投递中..."; }
        };
        System.out.println(anonymousTracker.getChannelName());

        // 2. 匿名内部类继承抽象类
        //    这就是你的 Shape 抽象类
        Shape anonymousShape = new Shape("匿名") {
            @Override
            public double area() { return 100.0; }
        };
        anonymousShape.describe();

        // 3. 匿名内部类作为方法参数
        //    最常见的用法：临时传一个实现
        processNotification(new Notification("系统", "测试消息") {
            @Override
            public void send() {
                System.out.println("自定义发送: " + getContent());
            }
        });
    }

    // 方法接受 Notification 类型参数 — 多态！
    public static void processNotification(Notification n) {
        n.preview();
        n.send();
    }
}
```

这段代码里：
- 匿名内部类 + 接口 = 不需要写 `implements Trackable` 的独立类
- 匿名内部类 + 抽象类 = 不需要写 `extends Shape` 的独立类
- 匿名内部类 + 方法参数 = 临时的、一次性实现

## 6. 常见误区

### 误区一：`new 接口()` 是 new 了接口

不对。

```java
Trackable t = new Trackable() { ... };
```

这里 `new` 的不是 `Trackable` 接口，而是 JVM 自动生成的**匿名类** `OuterClass$1.class`。这个匿名类实现了 `Trackable`。

### 误区二：匿名内部类可以替代所有具名类

不对。

匿名内部类适合**逻辑简单、只在一处使用**的场景。如果逻辑复杂或需要复用，还是应该写具名类。

判断标准：
- 类的方法超过 2-3 个 → 写具名类
- 类需要在多处复用 → 写具名类
- 逻辑简单 + 只用一次 → 用匿名内部类

### 误区三：成员内部类可以直接被外部独立创建

不对。

```java
// 错误！
University.Department dept = new University.Department("CS");

// 正确！需要先有外部类实例
University uni = new University();
University.Department dept = uni.new Department("CS");
```

成员内部类依附于外部类实例，不能脱离外部类单独存在。

### 误区四：静态嵌套类 = 加了 static 的成员内部类

不完全是。

静态嵌套类不持有外部类实例的引用，行为和独立的外部类几乎一样，只是放在另一个类内部作为命名空间。它不能访问外部类的非静态成员。

## 8. 一句话总结

- **成员内部类**：类中类，可以访问外部类的 private 成员，依赖外部类实例
- **静态嵌套类**：类中的静态类，本质是放在内部的独立类，不依赖外部类实例
- **局部内部类**：方法里的类，极少使用
- **匿名内部类**：new 的同时定义类体，是实现接口/抽象类的一次性快捷方式

> 内部类让代码的**组织粒度**更细——重要的类独立成文件，临时的辅助类放在使用它的地方。
