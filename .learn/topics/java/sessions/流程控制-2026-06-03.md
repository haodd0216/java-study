# 流程控制 — Learning Session

> **Date:** 2026-06-03
> **Topic:** java
> **Path:** 语言基础/流程控制
> **Level:** beginner+

---

## Positioning

「流程控制」是语言基础的第五站。前面学会了「程序由什么组成」（基本语法）、「名字怎么起」（标识符）、「数据怎么存」（变量与类型）、「数据怎么算」（运算符）——而流程控制回答的是最后一个核心问题：**程序以什么顺序执行。**

如果不控制流程，代码只会从第一行顺序执行到最后一行。流程控制让你能：根据条件分叉（if/switch）、重复执行（for/while）、中途跳出（break/continue/return）。

位置：语言基础 → 流程控制。它是从「能写线性代码」到「能写有逻辑的程序」的分水岭。

---

## Analogy

把程序执行想象成**开车上班的路**：

- 默认顺序执行 → 直行，一条语句接一条
- if/else → 路口看红绿灯，满足条件走 A，不满足走 B
- switch → 多岔路口，根据路牌指向不同出口
- for/while → 绕环岛，固定圈数或直到条件满足
- break → 走错了，立刻驶出
- continue → 这圈不绕了，跳到下一圈

## Core Mechanism

### 1. if / else if / else

```java
int score = 85;

if (score >= 90) {
    System.out.println("优秀");
} else if (score >= 80) {
    System.out.println("良好");        // ← 走这里
} else if (score >= 60) {
    System.out.println("及格");
} else {
    System.out.println("不及格");
}
```

关键细节：
- 条件**必须是 boolean**：`if (x = 5)` 在 Java 里会编译报错
- else 和最近的 if 配对，编译器只看花括号不看缩进
- 花括号可以省略但永远别省

### 2. switch

```java
switch (day) {
    case 1 -> name = "星期一";       // Java 14+ 箭头语法，自动不穿透
    case 2 -> name = "星期二";
    default -> name = "未知";
}
```

关键细节：
- fall-through 穿透是经典坑：传统写法没 break 会继续执行下一个 case
- 支持类型：byte, short, int, char, String, enum
- Java 14+ 的 switch 表达式更简洁安全

### 3. for（知道次数）

```java
// 传统 for
for (int i = 1; i <= 100; i++) { sum += i; }

// for-each
for (Income income : incomes) { total += income.getTax(); }
```

执行顺序：初始化(1次) → 判断 → 循环体 → 更新 → 判断 → ...

### 4. while（不知道次数，先判断）

```java
int n = 1;
while (n < 100) { n *= 2; }   // 128
```

先判断条件，条件为 true 才进循环体。一开始为 false 则一次不执行。

### 5. do-while（至少执行一次）

```java
do {
    sum += m;
    m++;
} while (m <= n);
```

循环体至少执行一次，条件判断在循环体之后。使用频率：for > while >>> do-while。

### 6. break —— 立即跳出最内层循环

```java
for (int i = 1; i <= 1000; i++) {
    if (i % 7 == 0 && i % 9 == 0) {
        System.out.println(i);  // 63
        break;                   // 找到就停
    }
}
```

带标签的 break 可以跳出指定层的循环（少用）。

### 7. continue —— 跳过本次，进入下一次

```java
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) continue;   // 偶数跳过
    System.out.print(i + " ");   // 1 3 5 7 9
}
```

### 8. return —— 终止方法

任何地方遇到 return，方法立即结束。不止返回值，更是一种强力流程控制。

## Code Example

```java
public class ScoreAnalyzer {
    public static void main(String[] args) {
        int[] scores = {85, 92, 58, 73, 66, 99, 45, 78};
        int excellent = 0, pass = 0, fail = 0;

        for (int score : scores) {                  // for-each 遍历
            if (score >= 90)       excellent++;     // if-else 分支
            else if (score >= 60)  pass++;
            else                   fail++;
        }

        // 查找第一个不及格
        int firstFailIndex = -1;
        for (int i = 0; i < scores.length; i++) {
            if (scores[i] < 60) { firstFailIndex = i; break; }
        }

        // switch 评语
        switch (scores[0] / 10) {
            case 10, 9 -> System.out.println("优秀");
            case 8, 7  -> System.out.println("良好");
            default    -> System.out.println("需努力");
        }
    }
}
```

## Common Misconceptions

1. 「if (x = 5) 也能用」→ Java 绝对不行，条件必须是 boolean
2. 「for-each 完全替代传统 for」→ 不能，需要索引/反向遍历/修改元素时用传统 for
3. 「switch 不如 if-else」→ 值匹配用 switch 更清晰、性能更好
4. 「break/continue 是 goto 的变体别用」→ 在限定的单层循环内完全可读，滥用标签跳出多层才是问题
5. 「while 和 for 只是写法不同」→ 语义不同：for = 遍历有边界范围，while = 等待条件成真

## Socratic Check

1. for 循环 `continue` 在 `i==3` 时 → 输出 0 1 2 4，跳过了 3 的 println
2. while 改 for：功能等价但语义不同，for 让读者一眼看到边界
3. switch 缺 break → fall-through 穿透，case 1 执行完会继续执行 case 2

---

## Quick Summary

- 三大流程：顺序、分支（if/switch）、循环（for/while/do-while）
- if 条件必须是 boolean，不要省花括号
- switch fall-through 是经典坑，Java 14+ 用箭头语法
- for 遍历已知次数，while 等待条件，do-while 至少一次
- break 跳循环，continue 跳本次，return 停方法
- 选择哪个循环 = 看你想给读者传递什么意图
