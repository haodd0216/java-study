# 多态

日期：2026-07-02
主题：Java / 面向对象编程 / 多态

## 1. 定位：多态解决什么问题？

你已经掌握了继承和方法重写。

回顾你刚写的 Shape 练习：

```java
class Shape {
    public double area() { return 0; }
    public void describe() {
        System.out.println("图形[" + name + "] 面积 = " + this.area());
    }
}

class Circle extends Shape {
    @Override
    public double area() { return Math.PI * radius * radius; }
}
```

你调用的是：

```java
circle.describe();
```

`describe()` 方法是父类 `Shape` 的，但里面有一句 `this.area()`。

当 `circle` 调用 `describe()` 时，`this.area()` 调用的是 `Circle` 里重写的 `area()`，而不是 `Shape` 里返回 0 的 `area()`。

这个现象就是多态：

> 同一个方法调用，根据对象的实际类型，表现出不同的行为。

多态更大的用途是：

> 用父类类型的变量来引用子类对象，代码可以统一处理多种子类，不需要一一判断类型。

比如把所有图形放进一个数组，统一调用 `describe()`：

```java
Shape[] shapes = { new Circle(3.0), new Rectangle(4.0, 5.0) };
for (Shape s : shapes) {
    s.describe();
}
```

`s` 是 `Shape` 类型，但实际上指向的是 `Circle` 或 `Rectangle`，调用 `describe()` 时会执行各自的逻辑。

这就是多态的核心价值：

> 写一次对父类的调用，自动适应所有子类。

## 2. 类比：遥控器和设备

可以把多态理解成"通用遥控器"。

家里有很多设备：电视、空调、音响。

它们都有一个"开机"按钮（共同行为）。

通用遥控器不管你插上的是哪种设备，按"开机"时就执行对应设备的开机逻辑：

- 电视：启动画面；
- 空调：开始制冷；
- 音响：播放音乐。

遥控器（父类引用）不需要知道你插的是哪种设备（子类对象），按按钮（调用方法）时自动执行对应逻辑。

这就是多态：

```text
遥控器 = 父类引用
设备   = 子类对象
开机   = 方法调用
执行哪种逻辑 = 运行时决定
```

## 3. 核心机制

多态的三个前提：

1. 继承（或实现接口）；
2. 方法重写；
3. 父类引用指向子类对象。

语法上，父类引用可以指向子类对象：

```java
Shape s = new Circle(3.0);
```

左边 `Shape` 是声明的类型，右边 `new Circle(3.0)` 是实际类型。

当你调用 `s.area()` 时，Java 在运行时看的是实际类型 `Circle`，所以调用的是 `Circle.area()`，不是 `Shape.area()`。

这叫**动态绑定**（Dynamic Dispatch）：

> 方法调用在编译时只检查父类有没有该方法，实际调用哪个版本在运行时根据对象的真实类型决定。

## 4. 代码例子

### 4.1 基本用法

```java
Shape s1 = new Circle(3.0);
Shape s2 = new Rectangle(4.0, 5.0);

s1.describe();  // 调用 Circle 的 area()
s2.describe();  // 调用 Rectangle 的重写 describe()
```

输出：

```text
图形[圆形] 面积 = 28.274333882308138
图形[矩形] 面积 = 20.0
宽=4.0 高=5.0
```

两个变量都是 `Shape` 类型，但行为不同。

### 4.2 数组统一处理

```java
Shape[] shapes = {
    new Circle(2.0),
    new Rectangle(3.0, 4.0),
    new Circle(5.0)
};

for (Shape s : shapes) {
    s.describe();
    System.out.println("---");
}
```

这段代码不需要知道每个元素是圆还是矩形，统一用 `Shape` 类型处理，各自调用自己的重写方法。

这就是多态最大的价值：

> 写代码时面向父类，运行时自动匹配子类行为。

### 4.3 计算所有图形面积之和

```java
double totalArea = 0;
for (Shape s : shapes) {
    totalArea += s.area();
}
System.out.println("总面积 = " + totalArea);
```

这段代码不关心每个 `s` 是哪种图形，直接调用 `area()`，Java 自动调用对应子类的实现。

以后如果加了 `Triangle`（三角形），只需要新建类并重写 `area()`，这段代码不用改，自动适配。

## 5. 父类引用的限制

父类引用指向子类对象时，只能访问父类里定义的方法：

```java
Shape s = new Circle(3.0);

s.area();       // 可以，Shape 有 area()
s.describe();   // 可以，Shape 有 describe()
s.getRadius();  // 编译错误，Shape 没有 getRadius()
```

`getRadius()` 是 `Circle` 特有的方法，用 `Shape` 类型的变量无法调用。

如果需要调用子类特有方法，需要向下转型：

```java
Shape s = new Circle(3.0);

if (s instanceof Circle) {
    Circle c = (Circle) s;
    System.out.println("半径：" + c.getRadius());
}
```

`instanceof` 用来检查对象实际是不是某种类型，`(Circle) s` 是强制转型。

但在实际设计里，经常需要向下转型说明多态用得不够充分。

好的设计应该让多态自动处理差异，而不是频繁转型。

## 6. 多态和你的 Shape 练习

你之前写的 `Shape.describe()` 里有这一行：

```java
System.out.println("图形[" + name + "] 面积 = " + this.area());
```

当时 `this` 是 `Circle` 或 `Rectangle` 对象，所以 `this.area()` 调用的是子类重写的版本。

这就是多态：

- `describe()` 是父类写的；
- `area()` 被子类重写；
- 父类的 `describe()` 调用 `area()` 时，自动调用子类版本。

这说明即使在父类方法内部，调用 `this.方法()` 也会走动态绑定，执行的是实际子类的版本。

这也是为什么同一个 `describe()` 方法，给圆形调用和给矩形调用时，面积数字不一样。

## 7. 完整例子：图形统计

```java
class Shape {
    private String name;

    public Shape(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }

    public double area() {
        return 0;
    }

    public void describe() {
        System.out.println("图形[" + name + "] 面积 = " + area());
    }
}

class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        super("圆形");
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}

class Rectangle extends Shape {
    private double width;
    private double height;

    public Rectangle(double width, double height) {
        super("矩形");
        this.width = width;
        this.height = height;
    }

    @Override
    public double area() {
        return width * height;
    }

    @Override
    public void describe() {
        super.describe();
        System.out.println("宽=" + width + " 高=" + height);
    }
}

public class Main {
    public static void main(String[] args) {
        Shape[] shapes = {
            new Circle(3.0),
            new Rectangle(4.0, 5.0),
            new Circle(1.5)
        };

        double total = 0;
        for (Shape s : shapes) {
            s.describe();
            total += s.area();
            System.out.println("---");
        }

        System.out.println("所有图形总面积 = " + total);
    }
}
```

输出：

```text
图形[圆形] 面积 = 28.274333882308138
---
图形[矩形] 面积 = 20.0
宽=4.0 高=5.0
---
图形[圆形] 面积 = 7.0685834705770345
---
所有图形总面积 = 55.342917353...
```

`main` 里只写了 `Shape` 类型的操作，不需要判断哪个是圆、哪个是矩形，多态自动处理。

## 8. 编译时类型 vs 运行时类型

```java
Shape s = new Circle(3.0);
```

- 编译时类型：`Shape`（编译器看到的是 `Shape`）；
- 运行时类型：`Circle`（程序运行时实际指向的对象）。

调用方法时：

- 编译器检查：`Shape` 有没有这个方法；
- 运行时执行：`Circle` 有没有重写这个方法，有就用 `Circle` 的，没有就用 `Shape` 的。

这叫**运行时多态**，也叫**动态绑定**。

字段没有多态，只有方法才会动态绑定。

## 9. 常见误区

### 误区一：多态只是"一个名字多个实现"

这个描述没错，但太抽象。

多态更具体的效果是：

> 父类引用调用方法时，运行时根据实际类型决定执行哪个版本。

### 误区二：父类引用调用子类特有方法

不行。

```java
Shape s = new Circle(3.0);
s.getRadius(); // 编译错误
```

父类引用只能调用父类里声明的方法。

要调用子类特有方法，需要向下转型或直接用子类类型声明变量。

### 误区三：`private` 方法也参与多态

不对。

`private` 方法不能被重写，因此不参与动态绑定。

子类写了一个同名的 `private` 方法，只是新定义了一个方法，不是重写。

### 误区四：字段也有多态

不对。

字段不参与动态绑定，访问的是编译时类型的字段版本。

只有方法才会动态绑定。

### 误区五：用了继承就一定有多态

不一定。

如果没有重写，父子类方法行为相同，谈不上多态带来的差异。

多态的效果来自重写后不同实现的存在。
