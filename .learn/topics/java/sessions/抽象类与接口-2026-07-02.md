# 抽象类与接口

日期：2026-07-02
主题：Java / 面向对象编程 / 抽象类与接口

## 1. 从 Shape 的问题出发

回顾你写的 `Shape` 类：

```java
class Shape {
    public double area() {
        return 0;  // 父类的 area() 返回 0，但这是什么意思？
    }
}
```

这里有一个设计上的别扭之处：

> 纯粹"图形"这个抽象概念不应该有面积。返回 0 只是一个"占位符"，不代表正确的行为。

如果有人直接 `new Shape("某个图形")` 然后调用 `area()`，得到 0——这没有意义。但现在的代码**允许**你直接 new Shape。

更好的设计是：

> 「Shape 不应该被直接实例化。它是一个**抽象概念**，只定义"图形应该有什么行为"，具体实现交给子类。」

这就是**抽象类**。

## 2. 抽象类（Abstract Class）

### 2.1 语法

```java
abstract class Shape {
    private String name;

    public Shape(String name) {
        this.name = name;
    }

    // 抽象方法：没有方法体，子类必须重写
    public abstract double area();

    // 具体方法：有方法体，子类继承
    public void describe() {
        System.out.println("图形[" + name + "] 面积 = " + area());
    }
}
```

关键字 `abstract` 有两个用法：

| 用法 | 含义 |
|------|------|
| `abstract class` | 这个类不能被 new，只能被继承 |
| `abstract` 方法 | 这个方法没有实现，子类必须重写 |

### 2.2 规则

**（1）抽象类不能实例化**

```java
Shape s = new Shape("图形");  // ✗ 编译错误！不能 new 抽象类
Shape s = new Circle(3.0);   // ✓ 可以用子类对象
```

**（2）抽象方法必须在子类中实现**

```java
abstract class Shape {
    public abstract double area();  // 只有声明，没有方法体
}

class Circle extends Shape {
    // 必须重写 area()，否则 Circle 也必须是 abstract class
    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}
```

如果子类不实现所有抽象方法，自己也必须声明为 `abstract`。

**（3）抽象类可以有构造方法**

虽然不能 `new Shape()`，但子类通过 `super()` 调用父类构造方法：

```java
abstract class Shape {
    private String name;

    public Shape(String name) {  // 构造方法，给子类的 super() 调用
        this.name = name;
    }
}

class Circle extends Shape {
    public Circle(double radius) {
        super("圆形");  // 调用抽象父类的构造方法 ✓
        this.radius = radius;
    }
}
```

**（4）抽象类可以有具体方法和字段**

```java
abstract class Shape {
    private String name;           // 字段
    public String getName() { ... } // 具体方法（有方法体）
    public abstract double area();  // 抽象方法（无方法体）
    public void describe() { ... }  // 具体方法（有方法体）
}
```

子类自动继承具体方法和字段，只需要实现抽象方法。

### 2.3 为什么要用抽象类

上面的例子如果对比两种写法：

```java
// 之前：普通父类
class Shape {
    public double area() { return 0; }  // 无意义的默认实现
}
// 问题：可能有人 new Shape() 然后 area() 返回 0，没意义

// 现在：抽象类
abstract class Shape {
    public abstract double area();       // 强制子类实现
}
// 好处：编译器帮你检查，每个子类都会实现 area()
```

> **抽象类 = 普通父类 + 编译期强制检查**：不允许不完整的对象存在。

### 2.4 苏格拉底式思考

```
问：如果用普通父类 + 方法返回默认值，能实现和抽象类一样的效果吗？
答：运行时能，但编译期不能。

问：区别在哪？
答：普通父类允许你忘记重写 area()，编译通过但运行时行为可能是错的。
   抽象类强制你必须重写，编译期就能发现问题。
```

这就是抽象类的价值：**将运行时错误变成编译期错误**。

## 3. 接口（Interface）

### 3.1 问题：抽象类够用了吗？

```java
abstract class Shape {
    public abstract double area();
}

class Circle extends Shape {
    @Override public double area() { ... }
}
```

字段 `area()` 所有图形都有。但如果有这样的需求：

- 有些图形可以**缩放**（resizable）
- 有些图形可以**旋转**（rotatable）
- 有些图形可以**序列化成 JSON**（serializable）

圆形要缩放、旋转、序列化；矩形只要缩放和序列化；三角形只要旋转。

怎么设计？如果用抽象类：

```java
abstract class Shape {
    public abstract double area();
    public abstract void scale(double factor);    // 缩放
    public abstract void rotate(double angle);    // 旋转
    public abstract String toJson();              // 序列化
}
```

问题来了：**不是所有图形都支持所有操作**！如果都放在父类，每个子类都要实现不需要的方法。

而且 Java **单继承**：一个类只能继承一个父类，不能同时继承 `Resizable` 和 `Rotatable`。

这就是**接口**要解决的问题。

### 3.2 接口的概念

> 接口是一种**能力契约**：定义一组方法，任何类都可以"签下这个契约"来获得这种能力。

类比：

```
抽象类 = 你是某种人（is-a 关系）
接口   = 你有某种能力（can-do 关系）

抽象类：程序员 是一种 人
接口：  程序员 会 编程、会 英语、会 开车
       "会编程""会英语""会开车"是可以叠加的能力标签
```

### 3.3 语法

**定义接口：**

```java
interface Drawable {
    void draw();  // 接口中的方法默认是 public abstract
}

interface Scalable {
    void scale(double factor);
}

interface Rotatable {
    void rotate(double angle);
}
```

**实现接口：**

```java
class Circle extends Shape implements Drawable, Scalable {
    @Override public double area() { ... }

    @Override public void draw() {
        System.out.println("画一个圆形");
    }

    @Override public void scale(double factor) {
        this.radius *= factor;
    }
}
```

一个类可以：
- 继承**一个**父类（单继承）
- 实现**多个**接口（多实现）

### 3.4 接口的关键特性

**（1）接口不能实例化**

```java
Drawable d = new Drawable();  // ✗ 编译错误
Drawable d = new Circle(3.0); // ✓ 实现类可以赋值给接口类型
```

**（2）接口中的方法默认是 public abstract**

```java
interface Drawable {
    void draw();  // 等价于 public abstract void draw();
}
```

**（3）接口中的字段默认是 public static final（常量）**

```java
interface Constants {
    double PI = 3.14159;  // 等价于 public static final double PI = 3.14159;
}
```

**（4）Java 8+ default 方法：接口可以有方法体**

```java
interface Drawable {
    void draw();  // 抽象方法，实现类必须重写

    default void drawWithBorder() {  // default 方法，有默认实现
        System.out.println("=== 开始绘制 ===");
        draw();
        System.out.println("=== 绘制完成 ===");
    }
}
```

`default` 方法给接口添加新方法而不破坏现有实现类（实现类可以重写，也可以直接用默认实现）。

### 3.5 接口 vs 多态

接口也是多态的基础：

```java
// 用接口类型引用实现类对象
Drawable d = new Circle(3.0);
d.draw();  // 多态：实际调用 Circle.draw()

Scalable s = new Rectangle(4, 5);
s.scale(2.0);  // 多态：实际调用 Rectangle.scale()
```

一个对象可以被多种类型的引用指向：

```java
Circle c = new Circle(3.0);

Shape shapeRef = c;     // 作为图形
Drawable drawRef = c;   // 作为可绘制对象
Scalable scaleRef = c;  // 作为可缩放对象

// 每种引用能看到的方法不同：
shapeRef.area();    // ✓（Shape 的方法）
shapeRef.draw();    // ✗（Shape 没有 draw）

drawRef.draw();     // ✓（Drawable 的方法）
drawRef.area();     // ✗（Drawable 没有 area）
```

## 4. 抽象类 vs 接口：什么时候用哪个？

| 维度 | 抽象类 | 接口 |
|------|--------|------|
| 关系 | "是什么"（is-a） | "能做什么"（can-do） |
| 继承 | 单继承（只能继承一个） | 多实现（可以实现多个） |
| 字段 | 可以有实例字段 | 只能有常量（static final） |
| 构造方法 | 有，给子类 super() 调用 | 没有 |
| 方法 | 可以有抽象方法 + 具体方法 | 抽象方法 + default 方法 |
| 访问修饰符 | 方法可以是任意访问级别 | 方法默认 public |
| 设计意图 | 提取子类共性，提供模板 | 定义行为契约，解耦 |

### 选择口诀

```
如果子类之间有共同的字段和默认行为 → 用抽象类
如果只需要定义"能做什么"的契约 → 用接口
如果既需要"是什么"又需要"能做什么" → 两者结合用
```

## 5. 结合你的 Shape 体系

把你现有的代码改造成更好的设计：

```java
// 抽象类：定义"图形是什么"
abstract class Shape {
    private String name;

    public Shape(String name) {
        this.name = name;
    }

    public String getName() { return name; }
    public abstract double area();     // 每个图形必须实现 area()
    public void describe() {
        System.out.println("图形[" + name + "] 面积 = " + area());
    }
}

// 接口：定义"图形能做什么"
interface Drawable {
    void draw();
}

interface Scalable {
    void scale(double factor);
}

// Circle：既是 Shape，又能 Draw，还能 Scale
class Circle extends Shape implements Drawable, Scalable {
    private double radius;

    public Circle(double radius) {
        super("圆形");
        this.radius = radius;
    }

    @Override public double area() {
        return Math.PI * radius * radius;
    }

    @Override public void draw() {
        System.out.println("   ◯");
    }

    @Override public void scale(double factor) {
        radius *= factor;
    }
}

// Rectangle：既是 Shape，又能 Draw，还能 Scale
class Rectangle extends Shape implements Drawable, Scalable {
    private double width, height;

    public Rectangle(double width, double height) {
        super("矩形");
        this.width = width;
        this.height = height;
    }

    @Override public double area() {
        return width * height;
    }

    @Override public void draw() {
        System.out.println("   ▭");
    }

    @Override public void scale(double factor) {
        width *= factor;
        height *= factor;
    }
}
```

现在多态可以用三种类型的引用来操作同一个对象：

```java
Circle circle = new Circle(3.0);

// 1. 作为 Shape（图形）— 调用 area、describe
Shape s = circle;
s.describe();

// 2. 作为 Drawable（可绘制）— 调用 draw
Drawable d = circle;
d.draw();

// 3. 作为 Scalable（可缩放）— 调用 scale
Scalable sc = circle;
sc.scale(2.0);
```

## 7. 一句话总结

- **抽象类**：让"不该被 new 的类"在编译期就被阻止，强制子类实现核心方法。
- **接口**：定义独立的能力契约，一个类可以实现多种能力，突破单继承限制。
- **两者结合**：抽象类表达"是什么"，接口表达"能做什么"，共同构建灵活的多态体系。

> 多态让你面向父类写代码；抽象类和接口让你写出更好的、更安全的"父类"。
