# 构造方法与封装的关系

日期：2026-06-18
主题：Java / 面向对象编程 / 封装 / 构造方法与封装的关系

## 1. 定位：这个概念解决什么问题？

你已经理解了封装的核心：字段通常设为 `private`，外部通过方法访问或修改，防止对象状态被随便破坏。

但这里会出现一个新问题：

> 如果对象一创建出来时，内部字段就是不合理的，那后面再封装也晚了。

例如：

```java
public class Student {
    private String id;
    private String name;
    private int age;
}
```

如果创建对象后没有给 `id`、`name`、`age` 设置合理值，这个对象可能处在“半成品”状态。

构造方法和封装的关系就是：

> 构造方法负责在对象创建时建立合法的初始状态；封装负责在对象创建后继续保护这个状态。

所以它们不是两个孤立知识点，而是配合使用的：

- 构造方法：对象出生时把关键字段设置好；
- `private` 字段：不让外部随便直接改；
- getter/setter 或业务方法：控制对象出生后的访问和变化。

## 2. 类比：办理身份证

可以把构造方法理解成“出生登记”或“办理身份证”。

一个人出生后，需要登记一些基本信息：姓名、出生日期、身份证号等。

这些信息中，有些可以以后改，比如昵称；有些通常不能随便改，比如身份证号。

对应到 Java 对象：

- 构造方法：创建对象时登记必要信息；
- `private` 字段：把这些信息保护起来；
- getter：允许外部查看；
- setter：只允许修改可以改变的字段；
- 不提供 setter：表示这个字段创建后不允许外部修改。

例如学生对象：

- `id` 学号：创建时确定，之后不能改；
- `name` 姓名：创建时确定，之后可以改；
- `age` 年龄：创建时确定，之后可以改，但必须合法。

## 3. 核心机制：构造方法让对象一出生就是合法的

先看一个不太好的写法：

```java
public class Student {
    private String id;
    private String name;
    private int age;

    public String getId() {
        return id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age < 0 || age > 150) {
            System.out.println("年龄不合法");
            return;
        }
        this.age = age;
    }
}
```

使用时：

```java
Student s = new Student();
System.out.println(s.getId());
System.out.println(s.getName());
System.out.println(s.getAge());
```

这时会得到：

- `id` 是 `null`；
- `name` 是 `null`；
- `age` 是 `0`。

虽然字段是 `private`，但对象刚创建出来时并不一定合理。

如果学生对象必须有学号和姓名，那么 `new Student()` 这种空对象就不应该被允许创建。

更好的写法是使用构造方法：

```java
public class Student {
    private String id;
    private String name;
    private int age;

    public Student(String id, String name, int age) {
        this.id = id;
        this.name = name;
        setAge(age);
    }

    public String getId() {
        return id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age < 0 || age > 150) {
            System.out.println("年龄不合法");
            return;
        }
        this.age = age;
    }
}
```

使用时：

```java
Student s = new Student("S001", "小明", 18);
```

这样一来，创建学生对象时就必须提供：

- 学号；
- 姓名；
- 年龄。

对象一创建出来，就有了基本完整的信息。

## 4. 构造方法和 setter 的配合

构造方法里有两种常见赋值方式。

第一种：直接赋值。

```java
public Student(String id, String name, int age) {
    this.id = id;
    this.name = name;
    this.age = age;
}
```

这种写法简单，但如果 `age` 传入 `-5`，对象就会保存非法年龄。

第二种：构造方法中调用已有的 setter。

```java
public Student(String id, String name, int age) {
    this.id = id;
    this.name = name;
    setAge(age);
}
```

这样 `setAge` 里的校验逻辑可以复用。

但要注意：初学阶段可以这样理解和使用；以后学到继承时，会补充一个细节：构造方法中调用可被子类重写的方法可能有风险。不过在当前没有继承的普通类里，调用自己的 setter 是可以帮助复用校验逻辑的。

## 5. 哪些字段适合通过构造方法设置？

通常有三类字段适合放进构造方法。

### 5.1 对象必须具备的字段

例如学生必须有学号和姓名：

```java
public Student(String id, String name) {
    this.id = id;
    this.name = name;
}
```

如果没有这些字段，对象就不像一个完整学生。

### 5.2 创建后不应该随便修改的字段

例如学号：

```java
private String id;

public Student(String id) {
    this.id = id;
}

public String getId() {
    return id;
}
```

这里没有 `setId`，表示学号只能在创建时设置，之后外部只能读取。

这就是你前面回答中提到的“创建对象时定义 ID，只能读取，不能修改”。

### 5.3 有默认初始状态的字段

例如账户余额初始为 0：

```java
public class BankAccount {
    private String accountNo;
    private double balance;

    public BankAccount(String accountNo) {
        this.accountNo = accountNo;
        this.balance = 0;
    }
}
```

外部创建账户时只需要提供账号，余额由类自己决定从 0 开始。

## 6. 只读字段：构造方法 + getter + 无 setter

封装里一个非常重要的组合是：

> 构造方法赋值，getter 读取，不提供 setter。

例如：

```java
public class Student {
    private String id;
    private String name;

    public Student(String id, String name) {
        this.id = id;
        this.name = name;
    }

    public String getId() {
        return id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}
```

含义是：

- `id`：创建时必须设置，以后只能读，不能改；
- `name`：创建时设置，以后可以通过 `setName` 修改。

这比所有字段都生成 getter/setter 更合理。

封装的目标不是“每个字段都有 get/set”，而是“每个字段只能以合理的方式被使用”。

## 7. 构造方法里的参数校验

如果创建对象时传入非法数据，构造方法也应该保护对象。

例如：

```java
public class Product {
    private String name;
    private int stock;
    private double price;

    public Product(String name, int stock, double price) {
        if (name == null || name.equals("")) {
            System.out.println("商品名不能为空");
            return;
        }
        if (stock < 0) {
            System.out.println("库存不能小于 0");
            return;
        }
        if (price < 0) {
            System.out.println("价格不能小于 0");
            return;
        }
        this.name = name;
        this.stock = stock;
        this.price = price;
    }
}
```

这段代码表达了一个思想：对象创建时就要尽量保证字段合法。

不过这里有一个初学者需要注意的点：

在构造方法里写 `return;` 只能提前结束构造方法，但对象仍然会被创建出来，只是字段可能还是默认值。

例如：

```java
Product p = new Product("", -1, -5);
```

对象 `p` 仍然存在，只是内部状态可能不完整。

更严格的做法是后面学习“异常”时使用 `throw` 阻止非法对象创建：

```java
public Product(String name, int stock, double price) {
    if (name == null || name.equals("")) {
        throw new IllegalArgumentException("商品名不能为空");
    }
    if (stock < 0) {
        throw new IllegalArgumentException("库存不能小于 0");
    }
    if (price < 0) {
        throw new IllegalArgumentException("价格不能小于 0");
    }
    this.name = name;
    this.stock = stock;
    this.price = price;
}
```

当前阶段你只要先理解：

> 构造方法不只是赋值，还可以负责对象初始状态的合法性。

异常机制之后再深入。

## 8. 默认构造方法和有参构造方法

如果一个类没有写任何构造方法，Java 会自动提供一个无参构造方法。

例如：

```java
public class Student {
    private String name;
}
```

可以这样创建：

```java
Student s = new Student();
```

但如果你自己写了有参构造方法：

```java
public class Student {
    private String name;

    public Student(String name) {
        this.name = name;
    }
}
```

Java 就不会再自动提供无参构造方法。

这时下面代码会报错：

```java
Student s = new Student();
```

只能这样：

```java
Student s = new Student("小明");
```

这其实也能帮助封装：你可以通过“不提供无参构造方法”，强制外部创建对象时传入必要信息。

## 9. 一个完整例子：银行账户

```java
public class BankAccount {
    private String accountNo;
    private double balance;

    public BankAccount(String accountNo) {
        this.accountNo = accountNo;
        this.balance = 0;
    }

    public String getAccountNo() {
        return accountNo;
    }

    public double getBalance() {
        return balance;
    }

    public void deposit(double amount) {
        if (amount <= 0) {
            System.out.println("存款金额必须大于 0");
            return;
        }
        balance += amount;
    }

    public void withdraw(double amount) {
        if (amount <= 0) {
            System.out.println("取款金额必须大于 0");
            return;
        }
        if (amount > balance) {
            System.out.println("余额不足");
            return;
        }
        balance -= amount;
    }
}
```

使用：

```java
public class Main {
    public static void main(String[] args) {
        BankAccount account = new BankAccount("A001");

        account.deposit(100);
        account.withdraw(30);

        System.out.println(account.getAccountNo());
        System.out.println(account.getBalance());
    }
}
```

这个类的封装设计是：

- `accountNo` 创建时传入，之后只能读取，不能修改；
- `balance` 创建时由类自己初始化为 0；
- 外部不能直接设置余额；
- 余额只能通过 `deposit` 和 `withdraw` 改变；
- 存款和取款方法内部负责校验金额。

这就是构造方法和封装的典型配合。

## 10. 常见误区

### 误区一：构造方法只是给字段赋值

不完全是。

构造方法更重要的作用是让对象创建出来时处于合理状态。

赋值只是手段，合法状态才是目标。

### 误区二：有了 setter，就不需要构造方法

不对。

如果一个对象必须有某些字段，应该在构造方法中要求传入。

否则对象可能先被创建成半成品，再等待别人调用 setter 补全。

这种半成品对象容易导致错误。

### 误区三：所有字段都应该放进构造方法

也不一定。

构造方法应该放“创建对象时必须知道”的信息。

有些字段可以有默认值，或者后续通过业务方法变化。

例如银行账户的余额可以由类自己初始化为 0，不一定让外部传入。

### 误区四：不提供 setter 就不能给字段赋值

不对。

字段可以在构造方法中赋值。

不提供 setter 只是表示：对象创建后，外部不能再改这个字段。

## 11. 判断设计是否合理

设计类时可以问自己：

1. 这个字段是不是对象创建时必须有？如果是，放进构造方法。
2. 这个字段创建后能不能被外部修改？如果不能，不提供 setter。
3. 这个字段修改时有没有规则？如果有，在 setter 或业务方法里校验。
4. 这个字段是不是应该由对象自己决定默认值？如果是，不必让外部传。
5. 是否存在“创建出来但信息不完整”的对象？如果有，考虑用构造方法避免。
