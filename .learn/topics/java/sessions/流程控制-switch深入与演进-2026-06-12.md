## Analogy

- 传统 switch（Java 1-13）= 密码解锁，每次手动 break，忘了就出大事
- 箭头 switch（Java 14+）= 指纹解锁，自动不穿透
- 模式匹配 switch（Java 17+ 预览）= 人脸识别，识别值的内部结构

## Core Mechanism

### 1. fall-through 的故意用法

多个 case 共享逻辑时，穿透简洁有力：

```java
switch (rank) {
    case 1:
    case 2:
    case 3: award = "金牌"; break;
    case 4:
    case 5:
    case 6: award = "银牌"; break;
}
```

**约定：故意穿透时加注释 `// fall through`**。

### 2. Java 14 箭头 switch

三处关键变化：
- **箭头 `->`** 替代冒号，自动不穿透，不用 break
- **switch 作为表达式**，直接返回值
- **多个 case 用逗号**分隔：`case 1, 3, 5 -> 31`

表达式 switch 必须穷尽所有可能（exhaustive），否则编译错误。

### 3. yield —— switch 表达式内部的 return

```java
String grade = switch (score / 10) {
    case 10, 9 -> "A";
    default -> {
        System.out.println("补考");
        yield "D";    // yield 只用于 switch 表达式的花括号内
    }
};
```

`yield` 不终止方法，只给 switch 返回值。它是上下文关键字。

### 4. enum + switch —— 天作之合

```java
enum Day { MONDAY, TUESDAY, ..., SUNDAY }

String feeling = switch (day) {
    case MONDAY -> "😫";
    case FRIDAY -> "😊";
    case SATURDAY, SUNDAY -> "🎉";
    default -> "😐";
};
```

好处：增减 enum 常量时 IDE 会标黄提醒补充 case。

### 5. 模式匹配 switch（Java 17+ 预览）

switch 不再只是值相等判断，而是能做类型 + 结构匹配：

```java
// 未来方向：
String result = switch (obj) {
    case Integer i      -> "整数: " + i;
    case String s       -> "字符串: " + s;
    case List<?> list   -> "列表: " + list.size();
    case null           -> "null";
    default             -> "未知";
};
```

**在预览阶段，生产环境稳定后再用。**

## Code Example

游戏段位计分系统，传统 vs 现代写法：

```java
// ✅ Java 14 箭头表达式
public static int bonusV2(Rank rank) {
    return switch (rank) {
        case BRONZE   -> 10;
        case SILVER   -> 25;
        case GOLD     -> 50;
        case PLATINUM -> 80;
        case DIAMOND  -> 120;
    };
}

// ✅ 分组 + yield
public static String tierMessage(Rank rank, int winRate) {
    return switch (rank) {
        case BRONZE, SILVER -> "新手区";
        case PLATINUM, DIAMOND -> {
            yield (winRate > 60 ? "顶尖" : "普通") + "高端玩家";
        }
        default -> "中坚力量";
    };
}
```

## Common Misconceptions

1. 「fall-through 是设计缺陷」→ 是特性不是缺陷，多个 case 共享逻辑是它的设计初衷。Java 14 箭头语法解决了意外穿透问题
2. 「switch 表达式的 default 总是必须的」→ enum 全量匹配时可省略，int/String 不能省略
3. 「yield 和 return 一回事」→ return 终止整个方法，yield 只给 switch 表达式返回值
4. 「switch 太老了别用」→ Java 14+ switch 表达式已非常安全简洁，模式匹配 switch 是未来方向
