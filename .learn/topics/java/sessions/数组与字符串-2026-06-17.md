## Analogy

- **数组** = 固定座位的火车。车厢号（索引）从 0 开始，每个座位只能坐指定类型的乘客。车开动后不能加减座位。
- **String** = 刻好的石碑。刻上去就改不了，必须重新刻。安全但折腾。
- **StringBuilder** = 白板。写了擦擦了写，最后抄到石碑上（toString()）。

## Core Mechanism — 数组

### 声明与初始化

```java
int[] arr = new int[5];           // 默认值 0
int[] arr2 = {1, 2, 3, 4, 5};    // 字面量
int[] arr3 = new int[]{1, 2, 3}; // 匿名数组
```

数组是对象，存在堆上。`int[]` 推荐写法（不是 `int arr[]`）。

### 核心 API

```java
arr.length;                          // 属性，不是方法！
Arrays.toString(arr);                // 打印
Arrays.sort(arr);                    // 原地排序
Arrays.copyOf(arr, newLength);       // 扩容
Arrays.equals(a1, a2);               // 内容比较
```

### 多维数组 = 数组的数组

Java 没有真正的二维数组——`int[][]` 是「一个数组，里面每个元素又是一个 int 数组」：

```java
int[][] matrix = {{1,2,3}, {4,5,6}, {7,8,9}};
matrix.length;      // 3 行
matrix[0].length;   // 3 列

// 每行长度可以不同！（不规则数组）
int[][] ragged = {{1,2}, {3,4,5,6}, {7}};
```

### 数组越界

```java
int[] arr = {10, 20, 30};
arr[3] = 40;  // ❌ ArrayIndexOutOfBoundsException —— 运行时炸
arr[-1] = 0;  // ❌ 同样炸，没有 Python 的负数索引
```

## Core Mechanism — String

### String 不可变

所有方法都不修改原对象，返回新 String。原因：
- **安全**：String 用于类加载、文件路径、网络连接——如果可变，攻击者可以篡改
- **线程安全**：不可变对象天然线程安全，不需要同步
- **String Pool 复用**：相同内容的字符串可以共享同一块内存

### == vs equals()

| 操作 | 比较什么 | 何时用 |
|------|---------|--------|
| `==` | 引用地址（是不是同一个对象） | 基本类型 |
| `.equals()` | 内容 | **比较字符串永远用这个** |

```java
String a = new String("Java");
String b = new String("Java");
a == b;         // false —— 不同对象
a.equals(b);    // true  —— 内容相同
```

### String Pool —— JVM 的字符串缓存

```java
String a = "hello";    // 字面量 → 进 String Pool
String b = "hello";    // Pool 中已有，直接复用
a == b;                // true! 同一个对象

String c = new String("hello");  // 强制在堆上新创建，绕过 Pool
a == c;                // false —— 不同对象
```

### 常用方法

`trim()`/`strip()`、`isEmpty()`/`isBlank()`、`contains()`、`indexOf()`、`startsWith()`/`endsWith()`、`substring()`、`replace()`/`replaceAll()`、`split()`、`String.join()`

### StringBuilder —— 循环拼接必备

```java
// ❌ 循环 + 每次 new 新对象，O(n²)
String s = ""; for (...) s += i;

// ✅ StringBuilder 内部可变数组，O(n)
StringBuilder sb = new StringBuilder(); for (...) sb.append(i);
```

| | String | StringBuilder | StringBuffer |
|---|--------|---------------|--------------|
| 可变 | ❌ | ✅ | ✅ |
| 线程安全 | ✅ | ❌ | ✅ synchronized |
| 默认选择 | 常量 | ✅ | 多线程 |

## Code Example

```java
int[] scores = {85, 92, 58, 73, 66, 99, 45, 78};
int sum = 0;
for (int s : scores) sum += s;

int[] sorted = Arrays.copyOf(scores, scores.length);
Arrays.sort(sorted);
int min = sorted[0], max = sorted[sorted.length - 1];

int[] grades = new int[5];
for (int s : scores) {
    if (s >= 90) grades[0]++; else if (s >= 80) grades[1]++;
    else if (s >= 70) grades[2]++; else if (s >= 60) grades[3]++;
    else grades[4]++;
}

StringBuilder report = new StringBuilder();
report.append("=== 成绩报告 ===\n")
     .append("平均: ").append((double) sum / scores.length).append("\n")
     .append("最高: ").append(max).append(", 最低: ").append(min);
```

## Common Misconceptions

1. 「arr.length 和 str.length() 只是历史包袱」→ 有设计原因：数组长度是创建时固定的属性；String 的 length() 是方法，因为内部 char 数组容量可能大于实际字符数
2. 「`"a"+"b"+"c"` 创建了 5 个对象」→ 编译时常量折叠，直接合并成 "abc"，1 个对象
3. 「字符串 == 大部分时候能用」→ 这个「大部分时候」最危险，永远用 equals()
4. 「数组过时了，用 ArrayList」→ 固定大小场景数组更快（无装箱、无扩容检查），集合框架内部也大量依赖数组
5. 「StringBuffer 比 StringBuilder 好因为线程安全」→ 局部变量无多线程风险，额外同步开销是浪费

## Socratic Check

1. "Hello" 能被修改吗？s = s + " World" 发生了什么？→ "Hello" 不可变，s+ 创建新对象 "Hello World"，s 指向新对象，原来的 "Hello" 等待 GC
2. new String("abc") vs "abc" — == 结果？→ false，new 强行走堆不进 Pool
3. 循环拼接字符串用什么？→ StringBuilder，+ 每次 new 对象形成 O(n²)
