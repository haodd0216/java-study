## 对照练习要求

已完成：

- `accountNo` 为私有字段，只提供 getter，不提供 setter。
- `ownerName` 为私有字段，提供 getter 和 setter。
- `balance` 为私有字段，只提供 getter，不提供 setter。
- 构造方法传入账号和持有人，余额由类初始化为 0。
- `deposit` 使用 `amount <= 0` 校验非法金额。
- `withdraw` 使用 `amount <= 0` 和余额不足校验。
- `main` 中覆盖了创建账户、打印信息、存款、取款、超额取款、非法存款、修改持有人和再次打印。

## 做得好的地方

1. `accountNo` 使用 `final`，很好地表达了账号创建后不可变。
2. `balance` 没有 setter，余额变化只能通过业务方法发生。
3. `withdraw` 对余额不足做了保护，账户余额保持为合法状态。
4. `main` 的测试流程已经完整覆盖了题目要求。

## 小问题

题目要求非法存款输出：

```text
存款金额必须大于0
```

当前输出是：

```text
存款金额需大于0
```

含义正确，只是和题目要求的文字不完全一致。如果希望严格对齐题目，可以把 `需` 改成 `必须`。
