## Positioning

“开发环境与运行模型”位于 Java 知识地图的 **语言基础** 分支，它回答的是：你写的一段 Java 代码，究竟靠哪些工具、经过哪些步骤，最后变成能运行的程序。

## Analogy

可以把 Java 开发想成“写菜谱、加工半成品、在厨房做菜”：

你写的 `.java` 文件像菜谱，是人能读懂的说明。`javac` 编译器像加工厂，它把菜谱加工成 `.class` 字节码，这是一种更标准、更适合机器执行的半成品。JVM 像厨房里的厨师，它拿到 `.class` 字节码后，在当前电脑上真正把程序跑起来。

这就是 Java 经典的一句话：**一次编写，到处运行**。不是说 `.java` 文件直接到处运行，而是说编译后的字节码可以交给不同操作系统上的 JVM，由 JVM 负责适配本地环境。

## Core Mechanism

学习 Java 开发环境，最重要的是先分清几个名字。

**JDK** 是 Java Development Kit，也就是 Java 开发工具包。你要写 Java、编译 Java、运行 Java，通常安装的就是 JDK。它里面包含编译器 `javac`、运行命令 `java`、标准类库，以及很多开发工具。

**JRE** 是 Java Runtime Environment，也就是 Java 运行环境。它主要负责“运行已经编译好的 Java 程序”。过去常说普通用户只需要 JRE，开发者需要 JDK。现在学习和开发时，你可以简单记住：**装 JDK 就够了，因为 JDK 里面已经包含运行 Java 所需的能力**。

**JVM** 是 Java Virtual Machine，也就是 Java 虚拟机。它不是一个你平时直接编写的东西，而是 Java 程序真正运行时依赖的执行环境。JVM 读取 `.class` 字节码，然后把它解释或编译成本机能执行的机器指令。

一段 Java 程序大致经历这几步：

1. 你写源代码：`Hello.java`
2. 用 `javac Hello.java` 编译
3. 得到字节码文件：`Hello.class`
4. 用 `java Hello` 启动程序
5. JVM 加载 `Hello.class`，找到 `main` 方法并执行

这里有两个容易混淆的点。

第一，`javac` 负责编译，`java` 负责运行。它们都是 JDK 里的命令，但职责不同。

第二，运行时写的是 `java Hello`，不是 `java Hello.class`。因为 `java` 命令需要的是“类名”，不是文件名。JVM 会根据类名去找对应的 `.class` 文件。

Java 程序入口通常长这样：

```java
public static void main(String[] args) {
    // 程序从这里开始执行
}
```

现在你不需要立刻把每个关键字都吃透，只要先记住：`main` 方法是 Java 应用程序常见的启动入口。等学到“类与对象”“方法”“static”时，这一行会慢慢完全展开。

## Code Example

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, Java!");
    }
}
```

如果文件名叫 `Hello.java`，可以在终端里这样运行：

```bash
javac Hello.java
java Hello
```

第一条命令会把 `Hello.java` 编译成 `Hello.class`。第二条命令会启动 JVM，让 JVM 加载 `Hello` 这个类，并执行里面的 `main` 方法。执行结果会在终端输出：

```text
Hello, Java!
```

这段代码里，`public class Hello` 定义了一个名叫 `Hello` 的类。Java 要求公共类名通常要和文件名一致，所以类叫 `Hello`，文件就叫 `Hello.java`。

`System.out.println` 是向控制台输出一行文本。现在先把它当成“打印一句话”的固定写法即可。

## Common Misconceptions

第一个常见误解是把 JDK、JRE、JVM 当成同一个东西。它们关系密切，但层级不同：JDK 是开发工具包，JRE 是运行环境，JVM 是运行字节码的虚拟机核心。

第二个常见误解是以为 Java 源代码会被直接运行。实际流程是：`.java` 先被编译成 `.class` 字节码，然后 JVM 运行字节码。

第三个常见误解是运行时带上 `.class` 后缀，比如 `java Hello.class`。这通常是不对的，应该使用类名：`java Hello`。

第四个常见误解是觉得“跨平台”意味着完全没有平台差异。更准确地说，Java 程序依赖 JVM 屏蔽大部分系统差异；不同平台需要安装对应平台的 JVM。

## Socratic Check

你可以顺着这两个问题想一下，不用急着答得很标准：

1. 如果你只有 `Hello.java`，但电脑没有安装 JDK，你能完成编译吗？直觉上应该不能，因为缺少 `javac` 这个编译工具。
2. 如果你已经有了 `Hello.class`，运行时为什么不是操作系统直接执行它？因为 `.class` 是 Java 字节码，真正理解它的是 JVM。

## Quick Summary

- JDK 用来开发 Java，里面包含编译、运行和标准类库等能力。
- `.java` 源代码先通过 `javac` 编译成 `.class` 字节码，再由 JVM 运行。
- Java 的跨平台能力主要来自 JVM：同一份字节码可以交给不同平台的 JVM 执行。
