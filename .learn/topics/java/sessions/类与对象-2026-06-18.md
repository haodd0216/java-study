## 一、定位：你在知识地图的什么位置？

你已经掌握了 Java 的**语言基础**——变量、运算符、流程控制、数组与字符串。到目前为止，你写的代码都是**面向过程**的：数据是数据，逻辑是逻辑，两者分离。

现在，你即将跨入 Java 最核心的领域：**面向对象编程（OOP）**。而「类与对象」是这扇大门的第一把钥匙。

```
你已走过的路 ─────────────────── 你在这里 ─────── 前方的路
变量 → 运算符 → 流程控制 → 数组字符串 → ★类与对象★ → 封装 → 继承 → 多态 → ...
```

**如果没有类与对象，你只能写小脚本；有了它，你才能构建真正的软件系统。**

## 二、类比：用你最熟悉的事物来理解

### 🏠 类的本质 = 建筑蓝图

想象你要建一个小区，里面有 100 栋房子。你会怎么做？

- **不会**：给每栋房子单独画一张图纸（重复劳动 100 次）
- **而是**：先画一张**标准蓝图**（类），然后按这张蓝图盖出 100 栋**实际的房子**（对象）

```
┌─────────────────────┐
│   类 (Class)         │
│   = 建筑蓝图         │
│                     │
│   定义了：           │
│   - 房子有什么属性    │  ← 字段 (fields)
│     (颜色、面积、楼层) │
│   - 房子能做什么      │  ← 方法 (methods)
│     (开门、关窗)      │
└─────────┬───────────┘
          │  new 关键字
          │  「按蓝图施工」
          ▼
┌─────────────────────┐
│   对象 (Object)      │
│   = 实际的房子        │
│                     │
│   房子A: 红色, 120㎡  │
│   房子B: 蓝色, 90㎡   │
│   房子C: 白色, 150㎡  │
└─────────────────────┘
```

### 🔑 关键洞见

| 概念 | 现实类比 | 在 Java 中 |
|------|---------|-----------|
| **类 (class)** | 蓝图 / 模具 / 物种定义 | `class House { }` |
| **对象 (object)** | 实物 / 产品 / 具体个体 | `new House()` |
| **字段 (field)** | 属性 / 特征 | `String color;` |
| **方法 (method)** | 行为 / 能做的事 | `void openDoor() { }` |
| **new 关键字** | 按蓝图施工 | `new House()` |
| **构造器 (constructor)** | 施工队 + 初始化 | `House(String c) { }` |

## 三、核心机制：逐层拆解

### 3.1 最小可运行的类

```java
// 定义一个类 —— 这是蓝图
class Dog {
    // 字段：狗有什么（状态/数据）
    String name;
    int age;

    // 方法：狗能做什么（行为）
    void bark() {
        System.out.println(name + " says: 汪汪！");
    }
}

// 使用类 —— 按蓝图创建对象
public class Main {
    public static void main(String[] args) {
        // ① 声明一个 Dog 类型的变量
        Dog myDog;

        // ② 用 new 创建对象（实例化）
        myDog = new Dog();

        // ③ 给字段赋值
        myDog.name = "旺财";
        myDog.age = 3;

        // ④ 调用方法
        myDog.bark();  // 输出: 旺财 says: 汪汪！
    }
}
```

**逐行解读**：

- `Dog myDog;` — 声明变量，此时 `myDog` 的值是 `null`，还没有实际对象
- `myDog = new Dog();` — **这是最关键的一行**。`new` 在堆内存中真正创建了一个 Dog 对象，然后把它的「地址」赋给 `myDog`
- `myDog.name = "旺财";` — 通过 `.` 操作符访问这个具体对象的字段
- `myDog.bark();` — 通过 `.` 操作符调用这个具体对象的方法

### 3.2 一张图看懂内存

```
栈 (Stack)                    堆 (Heap)
───────────                  ───────────
myDog ─────────────────────→ │  Dog对象    │
                              │  name: null→"旺财" │
myDog 存的是地址(引用)          │  age:  0→3  │
                              └────────────┘
```

**核心认知**：Java 中，对象变量存储的是**引用（地址）**，不是对象本身。这是理解 Java 内存模型的第一块基石。

### 3.3 引入构造器 — 让对象「出生就有名字」

前面的例子中，对象创建后字段是空的（`null` / `0`），需要手动赋值。这很麻烦且容易忘。**构造器**解决了这个问题：

```java
class Dog {
    String name;
    int age;

    // 构造器：名称与类名完全相同，没有返回值类型
    Dog(String n, int a) {
        name = n;   // 创建时就直接初始化
        age = a;
        System.out.println("一只狗诞生了！它叫" + name);
    }

    void bark() {
        System.out.println(name + "(" + age + "岁): 汪汪！");
    }
}

// 使用
Dog d1 = new Dog("旺财", 3);    // 输出: 一只狗诞生了！它叫旺财
Dog d2 = new Dog("来福", 1);    // 输出: 一只狗诞生了！它叫来福
d1.bark();  // 旺财(3岁): 汪汪！
d2.bark();  // 来福(1岁): 汪汪！
```

**构造器的三条铁律**：
1. 名称**必须**和类名完全一致
2. **没有**返回值类型（连 `void` 都不写）
3. 用 `new` 调用时自动执行

### 3.4 this 关键字 — 「我自己」

当参数名和字段名一样时，需要 `this` 来区分：

```java
class Dog {
    String name;
    int age;

    Dog(String name, int age) {   // 参数名和字段名相同
        this.name = name;         // this.name 是字段，name 是参数
        this.age = age;           // this.age 是字段，age 是参数
    }
}
```

**记住**：`this` 永远指向「当前这个对象自己」。在方法内部，谁调用这个方法，`this` 就是谁。

## 四、渐进式代码示例：从简单到完整

### 示例 1：银行账户（综合运用）

```java
class BankAccount {
    // 字段
    String owner;
    double balance;
    String accountNumber;

    // 构造器
    BankAccount(String owner, String accountNumber, double initialDeposit) {
        this.owner = owner;
        this.accountNumber = accountNumber;
        this.balance = initialDeposit;
    }

    // 存钱
    void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println(owner + " 存入 " + amount + " 元，余额: " + balance);
        } else {
            System.out.println("存款金额必须大于0");
        }
    }

    // 取钱
    void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println(owner + " 取出 " + amount + " 元，余额: " + balance);
        } else {
            System.out.println("余额不足或金额无效");
        }
    }

    // 显示信息
    void display() {
        System.out.println("==============");
        System.out.println("户主: " + owner);
        System.out.println("账号: " + accountNumber);
        System.out.println("余额: " + balance);
        System.out.println("==============");
    }
}

// 测试
public class BankDemo {
    public static void main(String[] args) {
        BankAccount a1 = new BankAccount("张三", "6222-0001", 1000.0);
        BankAccount a2 = new BankAccount("李四", "6222-0002", 500.0);

        a1.display();
        a1.deposit(200);     // 张三 存入 200 元，余额: 1200.0
        a1.withdraw(300);    // 张三 取出 300 元，余额: 900.0

        a2.display();
    }
}
```

> **观察**：`a1` 和 `a2` 各自独立，互不影响——这正是一个蓝图的多个实例。

### 示例 2：对象之间的交互

```java
class Student {
    String name;
    int score;

    Student(String name, int score) {
        this.name = name;
        this.score = score;
    }

    // 比较：判断自己是否比另一个学生分数高
    boolean isHigherThan(Student other) {
        return this.score > other.score;
        //     ↑ 自己的分数    ↑ 别人的分数
    }
}

// 测试
public class CompareDemo {
    public static void main(String[] args) {
        Student alice = new Student("Alice", 92);
        Student bob = new Student("Bob", 85);

        System.out.println(alice.isHigherThan(bob));  // true
        System.out.println(bob.isHigherThan(alice));   // false
    }
}
```

**注意这里**：`alice.isHigherThan(bob)` —
- 方法内部的 `this` = alice
- 参数 `other` = bob
- `this.score > other.score` = `92 > 85` = `true`

## 五、常见误解与陷阱 ⚠️

### 误解 1：「对象变量存的是对象本身」

```java
Dog a = new Dog("旺财", 3);
Dog b = a;    // b 和 a 指向**同一个**对象！

b.name = "改名了";
System.out.println(a.name);  // 输出 "改名了"，不是 "旺财"！
```

**真相**：`b = a` 复制的是**引用（地址）**，不是对象。两个变量指向同一个对象，通过任何一个修改，另一个也看到变化。

### 误解 2：「没写构造器 = 没有构造器」

```java
class Dog {
    String name;
}
// 这样也能 new Dog() ？！
Dog d = new Dog();  // 可以的！
```

**真相**：如果你不写任何构造器，Java 会自动提供一个**无参默认构造器**。但一旦你写了任何一个构造器（比如 `Dog(String n)`），默认构造器就消失了。

### 误解 3：「方法和函数是一回事」

不完全一样。**方法**是属于某个类/对象的函数。在 Java 中，你几乎不可能写一个不属于任何类的函数（这是 Java 纯面向对象的体现）。

### 误解 4：「字段不初始化就有值」

```java
class Example {
    int x;       // 默认 0
    String s;    // 默认 null
    boolean b;   // 默认 false
}
```

**真相**：类的字段有默认值（`0`, `null`, `false`），但**局部变量**没有！这是两者的重要区别：

```java
void test() {
    int x;           // 局部变量，未初始化
    System.out.println(x);  // 编译错误！必须先赋值
}
```

## 六、苏格拉底式检验 🧠

**Q1**：下面代码创建了几个对象？

```java
Dog d1 = new Dog("A", 1);
Dog d2 = new Dog("B", 2);
Dog d3 = d1;
```

**答案**：**2 个对象**。`new` 出现了两次，所以堆上只有两个 Dog 对象。`d3` 只是复制了 `d1` 的引用，指向第一个对象。

**Q2**：这段代码有什么问题？

```java
class Cat {
    String name;
    Cat(String name) {
        this.name = name;
    }
}
Cat c = new Cat();  // 这里能编译通过吗？
```

**答案**：**编译错误**。因为你定义了带参数的构造器 `Cat(String name)`，Java 不再自动提供无参构造器。`new Cat()` 找不到匹配的构造器。修复方法：手动添加 `Cat() { }` 或者调用 `new Cat("咪咪")`。

**Q3**：`this` 在什么情况下可以省略？什么情况下不能省略？

**答案**：
- **可以省略**：当方法内没有同名的局部变量/参数遮蔽字段时
- **不能省略**：当参数名和字段名相同时，`this.name = name` 中的 `this` 不能省，否则 `name = name` 是参数自己赋给自己

## 七、关键要点速查

| 要点 | 说明 |
|------|------|
| `class` | 定义类的关键字，类 = 蓝图 |
| `new` | 创建对象的关键字，在堆上分配内存 |
| 构造器 | 与类同名、无返回值，在 `new` 时自动调用 |
| `this` | 指向当前对象的引用 |
| 字段默认值 | `0`/`null`/`false`，仅针对类的字段 |
| 引用 vs 对象 | 变量存引用（地址），对象在堆上 |
