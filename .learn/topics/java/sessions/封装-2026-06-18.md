# 封装

日期：2026-06-18
主题：Java / 面向对象编程 / 封装

## 1. 定位：封装解决什么问题？

封装是面向对象编程的三大核心特征之一，另外两个是继承和多态。

在 Java 里，封装主要解决一个问题：**不要让外部代码随便直接操作对象内部的数据，而是通过对象提供的方法来安全地使用它。**

你已经学过“类与对象”，可以把类理解成一种模板，把对象理解成根据模板创建出来的具体事物。封装就是在这个模板里进一步规定：

- 哪些数据只能对象自己保存；
- 哪些操作可以暴露给外部使用；
- 外部想修改对象状态时，必须经过对象自己定义的规则。

所以封装不是简单地“把变量写进类里”，而是：

> 把数据和操作数据的方法放在一起，并通过访问控制保护内部状态。

## 2. 类比：自动售货机

可以把一个对象想象成一台自动售货机。

你作为使用者，不需要也不能直接打开机器，把里面的钱箱、货架、电路板随便改掉。你能做的是：

1. 投币；
2. 选择商品；
3. 等机器判断金额是否足够；
4. 机器自己决定是否出货、是否找零。

这里的“货架、钱箱、库存数量”就像对象的内部字段，应该隐藏起来。

“投币、选择商品、出货”就像对象对外提供的方法。

这样做的好处是：机器可以保证自己的状态不乱。例如库存不能变成负数，余额不能被外部随便改成 9999。

Java 的封装也是同样的思想：

- 字段通常设为 `private`；
- 外部通过 `public` 方法访问或修改；
- 方法内部可以加入校验逻辑。

## 3. 核心机制：private 字段 + public 方法

封装最常见的写法是：

```java
public class Student {
    private String name;
    private int age;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age < 0 || age > 150) {
            System.out.println("年龄不合法");
            return;
        }
        this.age = age;
    }
}
```

这段代码里：

- `private String name;` 表示 `name` 只能在 `Student` 类内部访问；
- `private int age;` 表示 `age` 也不能被外部直接修改；
- `getName()` / `getAge()` 用来读取字段；
- `setName()` / `setAge()` 用来修改字段；
- `setAge()` 里加入了规则：年龄不能小于 0，也不能大于 150。

外部使用时：

```java
public class Main {
    public static void main(String[] args) {
        Student s = new Student();

        s.setName("小明");
        s.setAge(18);

        System.out.println(s.getName());
        System.out.println(s.getAge());

        s.setAge(-5);
        System.out.println(s.getAge());
    }
}
```

如果字段是 `public`，外部可以这样写：

```java
s.age = -5;
```

这会让对象进入不合理状态。

但字段是 `private` 后，外部不能直接写：

```java
s.age = -5; // 编译错误
```

只能调用：

```java
s.setAge(-5);
```

而 `setAge` 方法内部可以拒绝这个非法值。

这就是封装的关键价值：**对象自己管理自己的数据。**

## 4. 访问修饰符：封装的边界工具

Java 里常见访问修饰符有四种访问级别：

| 修饰符 | 同一个类 | 同一个包 | 子类 | 任意位置 |
|---|---|---|---|---|
| `private` | 可以 | 不可以 | 不可以 | 不可以 |
| 默认不写 | 可以 | 可以 | 不可以跨包 | 不可以跨包 |
| `protected` | 可以 | 可以 | 可以 | 不可以任意访问 |
| `public` | 可以 | 可以 | 可以 | 可以 |

初学阶段先重点记住两个：

- `private`：保护内部字段；
- `public`：暴露给外部使用的方法。

常见封装原则是：

> 字段尽量 private，方法按需要 public。

也就是说，类内部的数据不要轻易公开；外部真正需要什么能力，再提供对应的方法。

## 5. this 关键字在封装中的作用

在 setter 方法里经常看到 `this`：

```java
public void setName(String name) {
    this.name = name;
}
```

这里有两个 `name`：

- 左边的 `this.name`：当前对象的成员变量；
- 右边的 `name`：方法参数。

如果写成：

```java
public void setName(String name) {
    name = name;
}
```

这只是把参数赋值给参数自己，成员变量不会变化。

所以 `this` 可以理解成：**当前这个对象本身。**

当局部变量和成员变量同名时，用 `this.成员变量` 来明确表示对象里的字段。

## 6. 封装不等于机械生成 getter/setter

很多初学者会以为：

> 只要字段 private，再给每个字段都生成 get/set，就是封装。

这只是一种形式，不一定是真正好的封装。

例如：

```java
public class BankAccount {
    private double balance;

    public double getBalance() {
        return balance;
    }

    public void setBalance(double balance) {
        this.balance = balance;
    }
}
```

这个类虽然用了 `private`，但外部仍然可以这样做：

```java
account.setBalance(-1000);
```

这对银行账户来说显然不合理。

更好的设计是，不直接提供 `setBalance`，而是提供符合业务含义的方法：

```java
public class BankAccount {
    private double balance;

    public double getBalance() {
        return balance;
    }

    public void deposit(double amount) {
        if (amount <= 0) {
            System.out.println("存款金额必须大于 0");
            return;
        }
        balance += amount;
    }

    public void withdraw(double amount) {
        if (amount <= 0) {
            System.out.println("取款金额必须大于 0");
            return;
        }
        if (amount > balance) {
            System.out.println("余额不足");
            return;
        }
        balance -= amount;
    }
}
```

这样外部不能随便把余额改成任意数字，只能通过“存款”和“取款”来改变余额。

这比单纯的 `setBalance` 更符合封装思想。

## 7. 封装带来的好处

### 7.1 防止非法数据

比如年龄不能是负数，余额不能随便改成负数，库存不能小于 0。

封装后，修改数据必须经过方法，方法里就能检查数据是否合法。

### 7.2 隐藏内部实现

外部只需要知道“怎么用”，不需要知道“里面怎么存”。

例如一个 `User` 类对外提供：

```java
public String getDisplayName() {
    return nickname + "(" + username + ")";
}
```

以后内部字段从 `nickname` 改成 `profileName`，只要 `getDisplayName()` 的返回结果不变，外部代码就不需要改。

### 7.3 降低代码之间的耦合

如果外部到处直接访问字段：

```java
user.name
user.age
user.phone
```

那么字段名、字段类型一改，很多地方都要跟着改。

如果外部通过方法使用对象：

```java
user.getName()
user.getAge()
```

类内部就有更多调整空间。

### 7.4 让对象更像“有行为的事物”

没有封装时，类容易变成一堆数据的容器。

有封装时，类会把数据和行为组织在一起。

例如 `BankAccount` 不只是保存 `balance`，它还知道如何 `deposit` 和 `withdraw`。

## 8. 常见误区

### 误区一：所有字段都必须有 getter 和 setter

不一定。

如果一个字段不希望外部修改，就不要提供 setter。

例如学生学号 `id` 创建后不应该随便改：

```java
public class Student {
    private String id;
    private String name;

    public Student(String id, String name) {
        this.id = id;
        this.name = name;
    }

    public String getId() {
        return id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}
```

这里有 `getId()`，但没有 `setId()`。

### 误区二：private 就等于安全

`private` 只是限制直接访问，不代表逻辑一定安全。

如果你提供了一个不加限制的 setter，外部还是能通过 setter 写入错误数据。

真正的封装要看：对象有没有保护自己的状态。

### 误区三：封装会让代码变麻烦

封装确实会多写一些方法，但它让代码更稳定。

小程序里直接 public 字段似乎更快；但程序一变大，直接暴露字段会让修改变得困难。

封装是在为后续维护留空间。

## 9. 一个完整例子：商品库存

```java
public class Product {
    private String name;
    private int stock;
    private double price;

    public Product(String name, int stock, double price) {
        this.name = name;
        setStock(stock);
        setPrice(price);
    }

    public String getName() {
        return name;
    }

    public int getStock() {
        return stock;
    }

    public double getPrice() {
        return price;
    }

    public void setPrice(double price) {
        if (price < 0) {
            System.out.println("价格不能小于 0");
            return;
        }
        this.price = price;
    }

    public void setStock(int stock) {
        if (stock < 0) {
            System.out.println("库存不能小于 0");
            return;
        }
        this.stock = stock;
    }

    public void sell(int count) {
        if (count <= 0) {
            System.out.println("销售数量必须大于 0");
            return;
        }
        if (count > stock) {
            System.out.println("库存不足");
            return;
        }
        stock -= count;
    }

    public void restock(int count) {
        if (count <= 0) {
            System.out.println("补货数量必须大于 0");
            return;
        }
        stock += count;
    }
}
```

使用：

```java
public class Main {
    public static void main(String[] args) {
        Product p = new Product("Java 练习本", 10, 19.9);

        p.sell(3);
        System.out.println(p.getStock());

        p.sell(20);
        System.out.println(p.getStock());

        p.restock(5);
        System.out.println(p.getStock());
    }
}
```

这个例子里，`stock` 不是让外部随便改，而是通过：

- `sell` 减少库存；
- `restock` 增加库存；
- `setStock` 设置初始或管理库存时也要校验。

这就是“把规则放进对象内部”。

## 10. 判断一个类是否封装良好

可以问自己几个问题：

1. 字段是不是不该公开的都设成了 `private`？
2. 外部是否只能通过方法来改变对象状态？
3. 修改状态的方法里有没有必要的规则校验？
4. 有没有不该提供的 setter？
5. 方法名字是否表达了业务动作，而不是只是在机械改字段？

如果答案大多是“是”，说明封装做得比较好。
