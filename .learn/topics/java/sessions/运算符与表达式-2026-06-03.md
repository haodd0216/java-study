## 定位

在知识地图中，运算符与表达式是 `语言基础` 的第 4 站，紧接在变量与数据类型之后。前一步学会了**怎么存数据**，这一步回答：**怎么对数据做运算、做比较、做判断**。没有运算符，变量只是一堆存了值的箱子，什么也干不了。

## 类比

把 Java 程序想成一个**计算器**：

- **变量** = 你存进计算器记忆键（M+）里的数字
- **运算符** = 计算器上的按钮：`+` `-` `×` `÷` `=` 还有 `>` `<` 比较键
- **表达式** = 你完整输入的那道算式，比如 `3 + 5 × 2`，它最终算出**一个结果**

就像计算器算式总是得到一个数，Java 里每个表达式最终都会**求值（evaluate）为一个值**。

## 核心机制

Java 的运算符分六大类，按日常使用频率排列：

---

### 3.1 算术运算符 — 做数学计算

| 运算符 | 含义 | 示例 | 结果 |
|--------|------|------|------|
| `+` | 加 | `5 + 3` | `8` |
| `-` | 减 | `5 - 3` | `2` |
| `*` | 乘 | `5 * 3` | `15` |
| `/` | 除 | `5 / 2` | `2`（整数截断！） |
| `%` | 取余（模） | `5 % 2` | `1`（余数） |
| `++` | 自增 1 | `i++` / `++i` | `i = i + 1` |
| `--` | 自减 1 | `i--` / `--i` | `i = i - 1` |

`/` 和 `%` 是两个新手容易踩坑的操作：

```java
int a = 10 / 3;    // 3（整数相除，截断）
int b = 10 % 3;    // 1（取余：10 ÷ 3 = 3 余 1）
```

`%` 最常见的用途是判断奇偶：`n % 2 == 0`（偶数），`n % 2 == 1`（奇数）。

**`++` 和 `--` 的前缀后缀区别**——面试和笔试高频考点：

```java
int i = 5;
int a = i++;   // 后缀：先用后加 → a = 5, i = 6
int b = ++i;   // 前缀：先加后用 → b = 7, i = 7（i 已经是 6 了）
```

规则只有一句话：**前缀先变后用，后缀先用后变**。

---

### 3.2 赋值运算符 — 把结果存起来

```java
int x = 10;      // 基本赋值
x += 5;          // 等价于 x = x + 5  → x = 15
x -= 3;          // 等价于 x = x - 3  → x = 12
x *= 2;          // 等价于 x = x * 2  → x = 24
x /= 4;          // 等价于 x = x / 4  → x = 6
x %= 4;          // 等价于 x = x % 4  → x = 2
```

`+=`、`-=` 这类是**语法糖**——写起来更短，编译器帮你还原成完整形式。但它还做了一件事：**隐含强制类型转换**：

```java
byte b = 10;
b = b + 5;       // 编译错误！b + 5 结果是 int，塞不进 byte
b += 5;          // 正确！+= 内部自动转了
```

---

### 3.3 比较运算符 — 判断真假

全部返回 `boolean`（`true` / `false`）：

| 运算符 | 含义 | 示例 |
|--------|------|------|
| `==` | 等于 | `a == b` |
| `!=` | 不等于 | `a != b` |
| `>` | 大于 | `a > b` |
| `<` | 小于 | `a < b` |
| `>=` | 大于等于 | `a >= b` |
| `<=` | 小于等于 | `a <= b` |

**最要命的陷阱**：`==` 是比较，`=` 是赋值。

```java
int score = 90;
if (score == 100) {   // 比较：score 等于 100 吗？
    // ...
}
if (score = 100) {    // 编译错误！Java 中 int 不能当 boolean 用（不像 C）
}
```

Java 好在——这个错误在 Java 里**直接编译报错**（不像 C/C++ 会静默通过），因为 `=` 赋值表达式返回 int，不是 boolean。

---

### 3.4 逻辑运算符 — 组合多个条件

| 运算符 | 含义 | 短路？ |
|--------|------|--------|
| `&&` | 逻辑与（并且） | 是 |
| `||` | 逻辑或（或者） | 是 |
| `!` | 逻辑非（取反） | — |

**短路求值**是 `&&` 和 `||` 最精妙的设计：

```java
// && 短路：左边为 false 时，右边根本不执行
int a = 0;
boolean result = (a != 0) && (10 / a > 1);  // 安全！右边被跳过了
//                          ↑ 如果执行会抛异常（除零），但这里不会

// || 短路：左边为 true 时，右边不执行
boolean ok = (a == 0) || (10 / a > 1);      // 安全！右边被跳过
```

还有 `&` 和 `|`（单个符号）——**不短路**，两边都会执行。绝大多数情况用 `&&` 和 `||` 就够了。

---

### 3.5 三元运算符 — 一行搞定 if-else

```java
// 语法：条件 ? 值1 : 值2
int max = (a > b) ? a : b;   // 如果 a>b 取 a，否则取 b

String grade = (score >= 60) ? "及格" : "不及格";
```

可以嵌套但别过度——嵌套三层三元比 if-else 还难读。

---

### 3.6 运算符优先级 — 谁先算？

从高到低，记住这个简化版就够了：

```
1. ()              括号最高
2. ++ -- !         一元
3. * / %           乘除取余
4. + -             加减
5. > < >= <=       比较
6. == !=           相等
7. &&              逻辑与
8. ||              逻辑或
9. = += -= *=      赋值（最低）
```

口诀：**括号 > 乘除 > 加减 > 比较 > 逻辑 > 赋值**。不确定时，**加括号**，既不丢人也不影响性能。

---

### 3.7 表达式中的类型提升

前面变量那节学过自动类型转换，在表达式里同样生效：

```java
byte b1 = 10, b2 = 20;
// byte b3 = b1 + b2;      // 编译错误！byte+byte 自动提升为 int
int b3 = b1 + b2;           // 正确

int i = 5;
double d = 2.5;
double result = i + d;      // int + double → double，结果 7.5
```

规则：**表达式中，所有 byte/short/char 先提升到 int；如果有一个 double，全部提升到 double**。

## 代码示例

一个综合示例，把上面各类运算符串起来：

```java
public class OperatorDemo {
    public static void main(String[] args) {
        // ---- 算术 ----
        int count = 10;
        int remainder = count % 3;           // 1
        count++;                              // count = 11

        // ---- 赋值 ----
        double price = 99.9;
        price *= 0.8;                         // 打八折，price = 79.92

        // ---- 比较 + 逻辑 ----
        int score = 85;
        boolean isGood = (score >= 80) && (score < 90);  // true
        boolean isExcellent = (score >= 90);             // false

        // ---- 三元 ----
        String level = (score >= 90) ? "优秀" 
                     : (score >= 80) ? "良好"   // 嵌套三元
                     : "一般";

        // ---- 输出 ----
        System.out.println("count = " + count);       // 11
        System.out.println("打折价: " + price);       // 79.92
        System.out.println("成绩良好? " + isGood);    // true
        System.out.println("等级: " + level);          // 良好
    }
}
```

输出：
```
count = 11
打折价: 79.92
成绩良好? true
等级: 良好
```

## 常见误区

**误区一："`=` 是比较，`==` 是赋值"** → 正好说反了。`=` 是赋值，`==` 是比较。记忆方法：一个等号往变量里"塞东西"，两个等号才是在"对比"。

**误区二："`i++` 和 `++i` 没区别"** → 单独使用时确实没区别（都是自增 1），但放在表达式里结果完全不同。`int a = i++;` 和 `int a = ++i;` 产生的 a 值不一样。

**误区三："`&&` 和 `&` 一样"** → `&&` 短路（左边 false 右边跳过），`&` 不短路（两边都执行）。`&&` 还能保护右边不抛异常。

**误区四："`10 / 3` 等于 3.333..."** → 整数相除等于 3。必须是 `10.0 / 3` 或 `(double)10 / 3`。

**误区五："`+` 只是加法"** → `+` 还能拼接字符串。`"Hello" + 42` 结果是 `"Hello42"`。如果一边是 String，另一边会被自动转成 String 再拼接。

## 快速总结

- 算术：`+ - * / % ++ --`，整数除法截断，`%` 取余
- 赋值：`= += -= *= /= %=`，`+=` 隐含强制类型转换
- 比较：`== != > < >= <=`，`==`是比较，`=`是赋值
- 逻辑：`&& || !`，`&&` 和 `||` 短路求值
- 三元：`条件 ? 值1 : 值2`，一行 if-else
- 优先级：括号 > 乘除 > 加减 > 比较 > 逻辑 > 赋值
- 表达式中的类型提升：byte/short/char 先升 int，有 double 全升 double
