## 定位

位运算是 `语言基础 → 运算符与表达式` 的深度分支。平常的 `+ - * /` 操作的是**十进制数值**（`5 + 3 = 8`），而位运算操作的是**二进制位**（`0101 & 0011 = 0001`）。它更底层，也因此更快——是 Java 中最接近硬件的操作。

## 类比

想象你面前有一排**拨动开关**，每个开关只能处于两个状态：开（1）或关（0）。一个 `int` 就是 32 个这样的开关排成一排。

- `&`（与）= **串联电路**：两个开关都开，灯泡才亮
- `|`（或）= **并联电路**：任意一个开关开，灯泡就亮
- `^`（异或）= **两路互斥开关**：两个开关状态不同，灯泡才亮
- `~`（取反）= 把每个开关**全部拨到对面**
- `<<` / `>>` = 把整排开关**集体向左/右滑动**

## 核心机制

### 按位与 `&` — 两位都是 1 才是 1

```
  1010  (10)
& 1100  (12)
  ----
  1000  (8)
```

```java
int a = 10 & 12;  // 8
```

**实战用途**：判断奇偶。`n & 1` 为 0 → 偶数，为 1 → 奇数。比 `n % 2` 快。

```java
boolean isEven = (n & 1) == 0;  // 偶数
```

### 按位或 `|` — 任意一位是 1 就是 1

```
  1010  (10)
| 1100  (12)
  ----
  1110  (14)
```

```java
int a = 10 | 12;  // 14
```

**实战用途**：权限组合。比如 `READ = 1, WRITE = 2, EXECUTE = 4`，`READ | WRITE = 3`。

### 按位异或 `^` — 不同为 1，相同为 0

```
  1010  (10)
^ 1100  (12)
  ----
  0110  (6)
```

异或有两个神奇性质：
- **任何数异或自己得 0**：`a ^ a = 0`
- **任何数异或 0 得自己**：`a ^ 0 = a`

**实战用途**：交换两个变量，不需要第三个变量：

```java
int a = 5, b = 9;
a = a ^ b;
b = a ^ b;  // b 变成原来的 a
a = a ^ b;  // a 变成原来的 b
// 现在 a=9, b=5
```

**实战用途 2**：找出数组中唯一出现奇数次的元素（经典面试题）：

```java
int[] arr = {1, 2, 3, 2, 1};
int result = 0;
for (int n : arr) {
    result ^= n;  // 成对的抵消为 0，最后剩下 3
}
// result = 3
```

### 按位取反 `~` — 0 变 1，1 变 0

```java
int a = ~10;
//  0000...1010  (10)
// ~
//  1111...0101  (-11)
```

结果通常是 `-(n+1)`：`~10 = -11`。日常用得最少。

### 左移 `<<` — 等价于 × 2ⁿ

```
0010  (2)
 << 2   左移两位
1000  (8)   = 2 × 2²
```

```java
int a = 2 << 2;   // 8
int b = 1 << 5;   // 32（相当于 Math.pow(2, 5)）
```

**实战用途**：快速计算 2 的幂。`1 << n` = 2ⁿ。

### 右移 `>>`（算术）和 `>>>`（逻辑）

```java
// 正数的 >> 和 >>> 效果一样
int a = 8 >> 2;    // 2（8 / 4）
int b = 8 >>> 2;   // 2

// 负数的区别：>> 保留符号（补 1），>>> 不保留（补 0）
int c = -8 >> 2;   // -2（还是负数）
int d = -8 >>> 2;  // 1073741822（变成正数了！）
```

`>>` 是算术右移（保留符号），`>>>` 是逻辑右移（高位永远补 0，结果非负）。

**实战用途**：快速除以 2 的幂。`n >> 1` = `n / 2`，快得多。

## 代码示例

```java
public class BitwiseDemo {
    public static void main(String[] args) {
        int a = 10;  // 二进制: 1010
        int b = 12;  // 二进制: 1100

        System.out.println("a & b  = " + (a & b));   // 8   (1000)
        System.out.println("a | b  = " + (a | b));   // 14  (1110)
        System.out.println("a ^ b  = " + (a ^ b));   // 6   (0110)
        System.out.println("~a     = " + (~a));      // -11

        System.out.println("1 << 4 = " + (1 << 4));  // 16  (10000)
        System.out.println("16 >> 2 = " + (16 >> 2)); // 4

        // 实战：奇偶判断
        int n = 7;
        System.out.println(n + "是" + ((n & 1) == 1 ? "奇数" : "偶数"));

        // 实战：权限组合
        int PERM_READ  = 1;    // 001
        int PERM_WRITE = 2;    // 010
        int PERM_EXEC  = 4;    // 100
        int myPerm = PERM_READ | PERM_WRITE;          // 011 = 3
        boolean canRead  = (myPerm & PERM_READ)  != 0;  // true
        boolean canExec  = (myPerm & PERM_EXEC)  != 0;  // false
        System.out.println("可读: " + canRead + ", 可执行: " + canExec);
    }
}
```

**代码走读**：
- `a & b`：10(1010) 和 12(1100) 逐位与，只有第 4 位（从右数）都是 1 → 1000 = 8
- `a | b`：逐位或，只要有一位是 1 就取 1 → 1110 = 14
- `a ^ b`：逐位异或，不同的位取 1 → 0110 = 6
- `1 << 4`：1 的二进制是 0001，左移 4 位 → 10000 = 16
- 权限检查：`myPerm & PERM_READ` 如果结果不是 0，说明 READ 位被设置了

## 常见误区

**误区一："`&` 和 `&&` 可以互换"**
→ 不能。`&` 是按位运算符，操作二进制位；`&&` 是逻辑运算符，操作 boolean 值。`true & false` 虽然能编译（两边都算，不短路），但 `5 & 3` 是位运算，`5 && 3` 直接报错。

**误区二："`>>` 和 `>>>` 没区别"**
→ 正数时确实一样。负数时天差地别。`>>` 保留符号位（补 1），`>>>` 高位补 0。`-1 >> 1` 还是 `-1`，`-1 >>> 1` 变成 `2147483647`。

**误区三："左移就是乘以 2ⁿ，一定比乘法快"**
→ 对 JVM 来说，现代 JIT 编译器会把 `n * 8` 自动优化成 `n << 3`。所以**可读性优先**——写 `* 8` 比 `<< 3` 更清晰，性能没差。

**误区四："位运算是炫技，日常用不到"**
→ 权限系统、网络协议解析、哈希算法、BitSet、布隆过滤器……底层到处都是位运算。理解了它能看懂很多源码。

## 快速总结

- `&` 位与、`|` 位或、`^` 位异或、`~` 取反、`<<` 左移、`>>` 算术右移、`>>>` 逻辑右移
- `n & 1` 判断奇偶；`1 << n` 计算 2ⁿ；`n >> 1` 快速除以 2
- 异或 `^` 性质：`a ^ a = 0`，`a ^ 0 = a`，可用于交换变量和找唯一出现数
- `>>` 保留符号位，`>>>` 高位补 0，负数时差异巨大
- 日常写 `* 8` 而非 `<< 3`，JIT 会自动优化；位运算留给权限、哈希等底层场景
