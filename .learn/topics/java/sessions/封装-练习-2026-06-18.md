# 封装练习反馈

日期：2026-06-18
主题：Java / 面向对象编程 / 封装
练习文件：`com/fuhao/encapsulation/BankAccountPractice.java`

## 完成情况

本次实现了 `BankAccount` 类，包含：

- `private final String accountNo`
- `private String ownerName`
- `private double balance`
- 构造方法传入账号和持有人姓名
- `accountNo` 只读，无 setter
- `ownerName` 可读可写，并在 setter 中校验空值
- `balance` 只读，无 setter
- 使用 `deposit` 和 `withdraw` 改变余额

## 验证结果

执行命令：

```bash
javac "com/fuhao/encapsulation/BankAccountPractice.java" && java com.fuhao.encapsulation.BankAccountPractice
```

编译通过。

运行输出：

```text
9527
张三
5000.0
4500.0
Exception in thread "main" java.lang.IllegalArgumentException: 取款金额超过余额
```

程序在超额取款时抛出异常并中断。

## 做得好的地方

1. `accountNo` 使用了 `private final`，比题目要求更进一步表达了“创建后不可变”。
2. 没有提供 `setAccountNo`，符合只读字段设计。
3. 没有提供 `setBalance`，余额只能通过 `deposit` 和 `withdraw` 改变，符合封装思想。
4. 构造方法里调用 `setOwnerName(ownerName)`，复用了姓名校验逻辑。
5. `withdraw` 中检查余额不足，保护了账户余额不会变成负数。

## 需要调整的地方

1. 题目要求非法存款金额和非法取款金额打印指定文字，而当前实现使用 `throw new IllegalArgumentException(...)`。异常写法更严格，但和本次练习的预期输出不完全一致。
2. `deposit` 和 `withdraw` 当前判断的是 `amount < 0`，但题目要求金额必须大于 0，所以应该拒绝 `amount <= 0`。
3. `main` 方法还没有测试修改持有人为 `小王`，也没有再次打印持有人和余额。
4. 构造方法目前没有写 `public`，在同包内可以使用；如果严格按题目签名，应写成 `public BankAccount(String accountNo, String ownerName)`。

## 建议修正方向

当前阶段建议先按题目要求使用打印并 `return`：

```java
if (amount <= 0) {
    System.out.println("存款金额必须大于0");
    return;
}
```

等学到异常之后，再把非法输入改成 `throw new IllegalArgumentException(...)` 会更自然。
