## Analogy

- 传统 for = 手动挡汽车：能精确控制档位（索引）、随时变道、倒车，功能最强但你得操心
- for-each = 自动挡汽车：省心但只能正向开，大多数场景最该用
- while = 等公交：不知道车什么时候来，站着等条件满足

## Core Mechanism

### 1. 字节码真相

- for 和 while：对数组几乎一样的字节码，都是基于索引的 goto 指令
- for-each 对数组：编译器用专用指令（`arraylength` + 位置读），避免每次索引计算
- for-each 对集合：翻译成 Iterator 模式（`iterator() → hasNext() → next()`）

结论：遍历数组用 for-each，最安全也最快。

### 2. Iterator 原理

for-each 遍历集合 = Iterator 的语法糖：

```java
for (String s : list) { ... }
// 等价于
for (Iterator<String> it = list.iterator(); it.hasNext(); ) {
    String s = it.next();
}
```

**致命限制：遍历时不能直接修改集合结构**（`list.remove()` 抛 `ConcurrentModificationException`），必须用 `Iterator.remove()`。

### 3. 不能用 for-each 的三种场景

- 需要索引（比较相邻元素）
- 反向遍历
- 修改数组元素值（x 是副本）

### 4. 循环不变量外提

- `list.size()` 等简单 getter → JIT 会内联，不用手动提
- `Math.sqrt()`、字符串拼接、文件读取等 → **一定提出来**
- 不确定时 → 提出来更安全

### 5. 决策矩阵

| 场景 | 推荐 |
|------|------|
| 遍历数组/集合每个元素 | for-each |
| 需要索引/相邻元素 | 传统 for |
| 修改数组元素值 | 传统 for |
| 反向遍历 | 传统 for |
| 遍历时删除元素 | while + Iterator |
| 不知道循环次数 | while |
| 至少执行一次 | do-while |
| filter/map 流式处理 | Stream API |

## Code Example

```java
// ✅ 遍历 → for-each
for (int x : arr) { sum += x; }

// ✅ 需要索引 → 传统 for
for (int i = 0; i < arr.length - 1; i++) {
    if (arr[i] > arr[i+1]) return i;
}

// ✅ 遍历中删除 → while + Iterator
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (it.next().isEmpty()) it.remove();
}

// ✅ 循环不变量外提
final double SQRT2 = Math.sqrt(2);
for (int i = 0; i < arr.length; i++) { result[i] = arr[i] * SQRT2; }
```

## Common Misconceptions

1. 「while 比 for 快」→ 字节码等价，选哪个看语义
2. 「for-each 很慢」→ 对数组比传统 for 还快，对集合 Iterator 开销微乎其微
3. 「size() 必须提到循环外」→ ArrayList 不需要（JIT 内联），LinkedList 有意义
4. 「手动循环展开是优化」→ 永远不要，JIT 自动做

## Socratic Check

1. for-each 遍历数组 vs 传统 for？→ for-each 用专用指令略快，且更安全无越界风险
2. `for(String s : list) { list.remove(s); }` 为什么抛异常？→ for-each = Iterator，list.remove() 绕过 Iterator，检测到外部修改抛 ConcurrentModificationException
3. LinkedList 遍历删除前 k 个？→ while + Iterator，因为 get(i) 是 O(n)，Iterator 是 O(1)
