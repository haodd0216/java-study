## Positioning

「基本语法与程序结构」在知识地图中处于 **语言基础** 分支的第二站。它回答一个核心问题：**一个 Java 程序由哪些"积木块"组成？这些积木块之间是什么关系？**

如果说「开发环境与运行模型」回答的是「怎么写、怎么跑」，那这节回答的就是「程序长什么样、各部件怎么搭在一起」。

## Analogy

把 Java 程序想象成 **一幢大楼**：

- **`.java` 文件** = 大楼的每一层
- **类（class）** = 这一层的平面图——定义了这层有什么房间、怎么布局
- **方法（method）** = 每个房间——里面放着具体的家具和执行逻辑
- **语句（statement）** = 家具的每一块板、每一颗螺丝——最小的执行单元
- **包（package）** = 楼层编号——让不同楼层之间能有序地找到彼此
- **`main` 方法** = 大楼的正门入口——你每次进这幢楼，都必须从这个门进去

一幢大楼（一个 Java 程序），从入口进去，按楼层编号找到对应的楼层，进入某个房间，按家具的摆放顺序一步步执行。

## Core Mechanism

### 1. 程序的最小完整结构

一个能跑起来的 Java 程序，至少有这三个层次：

```
源文件 (.java)
  └── 类 (class)
        └── main 方法 (method)
              └── 语句 (statements)
```

写成代码就是：

```java
// 1. 源文件: Main.java
public class Main {                    // 2. 类定义

    public static void main(String[] args) {   // 3. main 方法入口
        System.out.println("Hello");           // 4. 执行语句
    }
}
```

从外到内，逐层拆开：

### 2. 源文件规则

- **文件名必须与 public 类名完全一致**：`public class Main` 必须保存在 `Main.java`
- **一个文件最多有一个 public 类**，但可以有多个非 public 类（先知道，后面会遇到）
- **Java 是大小写敏感的**：`Main` 和 `main` 是两个东西

### 3. 类（class）——程序的基本组织单元

Java 是纯面向对象语言，**所有代码都必须写在类里面**，没有例外：

```java
public class Main {
    // 这个花括号 { } 之间的所有代码，都属于 Main 这个类
}
```

类名通常用**大驼峰命名**（PascalCase），每个单词首字母大写。

### 4. main 方法——程序的入口

JVM 启动程序时，会去找这个签名的方法：

```java
public static void main(String[] args)
```

这行拆开看：

| 关键字 | 含义 |
|---|---|
| `public` | 公开的，JVM 从外部调用它 |
| `static` | 静态的，不需要先创建对象就能调用 |
| `void` | 这个方法不返回任何值 |
| `main` | 约定好的入口方法名，JVM 专门找它 |
| `String[] args` | 命令行参数，一个字符串数组 |

> ⚠️ 这个签名必须**一字不差**，即使是 `Main` 或 `MAIN` 也不行，必须是 `main`。

### 5. 语句 —— 程序的最小执行单元

语句是方法体内真正"干活"的东西，**每条语句以分号 `;` 结尾**：

```java
System.out.println("Hello");   // 输出语句
int age = 25;                  // 声明 + 赋值语句
age = age + 1;                 // 计算赋值语句
```

分号在 Java 里就像句号在中文里——它告诉编译器"一句话说完了"。

### 6. 注释 —— 写给人看的说明

```java
// 这是单行注释，从 // 到行尾

/*
 * 这是多行注释
 * 可以跨越多行
 */

/**
 * 这是文档注释（Javadoc）
 * 可以用 javadoc 工具生成 API 文档
 */
```

### 7. 代码块与缩进

花括号 `{ }` 定义了一个**代码块**，它划定了作用范围。Java 编译器不关心缩进（不像 Python），但**约定上必须缩进**，这是给人类看的：

```java
public class Main {             // 类体开始
    public static void main(String[] args) {   // 方法体开始
        if (true) {             // if 体开始
            System.out.println("缩进 3 层");
        }                       // if 体结束
    }                           // 方法体结束
}                               // 类体结束
```

## Code Example

下面是一个综合示例，把上述概念串起来：

```java
/**
 * 一个简单的加法计算器
 * 文件名: Calculator.java
 */
public class Calculator {                          // 类定义

    // main 方法：程序入口
    public static void main(String[] args) {       // 方法开始
        // 声明两个变量
        int a = 10;
        int b = 20;

        // 计算并输出结果
        int result = a + b;
        System.out.println("结果是: " + result);   // 输出: 结果是: 30
    }                                              // 方法结束

}                                                  // 类结束
```

运行方式（假设已安装 JDK）：
```bash
javac Calculator.java    # 编译 → 生成 Calculator.class
java Calculator          # 运行 → 输出: 结果是: 30
```

这个简单程序包含了：源文件、类定义、`main` 方法入口、变量声明、运算、方法调用、输出语句——Java 程序结构的所有基本元素。

## Common Misconceptions

**误解一：「程序结构不重要，先把功能写出来就行」**
→ 结构是 Java 的语法骨架。不理解「代码必须写在类的方法里」，你会写出挂在外面的语句然后编译报错，却不知道为什么。

**误解二：「文件名和类名不同只是一件小事」**
→ 它们是强绑定关系。`public class Hello` 写在 `Main.java` 里会直接编译失败。这不是风格问题，是语法规则。

**误解三：「main 方法的签名可以随便调整」**
→ 不能。写成 `public void main()`、`static void main(String[] args)`、`public static void Main(String[] args)` 都会导致 JVM 找不到入口，程序无法启动。

**误解四：「Java 和 Python/C 一样，缩进影响执行逻辑」**
→ Java 的缩进纯粹是给人看的，编译器只看花括号和分号。但这不意味着可以乱缩进——代码是写给未来的你和同事看的。

**误解五：「学了后面 OOP 的内容，前面的语法基础已经不重要了」**
→ 已有的 OOP 知识（继承、多态等都是高级用法）可以用基础语法来重新审视：每个 `.java` 文件的结构、每个类的边界——理解这些会让 OOP 代码更清晰。

## Socratic Check

1. 为什么 `System.out.println` 不能写在类的外面？→ 因为 Java 要求所有代码都必须写在类里面。
2. 如果文件名是 `App.java`，但里面的 public 类是 `Application`，会发生什么？→ 编译错误。public 类名必须与文件名一致。
3. 去掉 `main` 方法的 `static`，程序运行时会怎样？→ JVM 找不到入口，抛出 `NoSuchMethodError: main`。

---

## Quick Summary

- Java 程序结构：源文件 > 类 > main 方法 > 语句
- 文件名必须与 public 类名完全一致（大小写也一致）
- 所有执行代码必须在类的方法内部
- main 方法签名固定为 `public static void main(String[] args)`
- 每条语句以分号 `;` 结尾
- 花括号定义代码块，缩进是人看的（编译器不关心）
