## Analogy

把 Java 语言想像成一个**国家**：

- **关键字** = 法律条文里固定的术语词汇（「应当」「不得」），你不能重新定义它们
- **标识符** = 每个公民的姓名，法律不预先占满，你有自由但受规则约束
- **命名规范** = 社会习俗（姓在前名在后、制服统一），不遵守也能活，但社会运转效率低

## Core Mechanism

### 1. 标识符合法规则

```
✅ 可以包含：字母 A-Z a-z、数字 0-9、下划线 _、美元符 $
✅ 不能以数字开头
✅ 不能是关键字（及 true/false/null）
```

```java
// ✅ 合法
int age, _count, $price, 年龄, player1;

// ❌ 非法
int 1player;     // 数字开头
int my-age;      // 连字符被当减号
int class;       // 关键字
int my variable; // 空格
```

### 2. 命名规范（convention）—— 编译器不管但人管

| 元素 | 规范 | 示例 |
|------|------|------|
| 类名/接口名 | PascalCase 大驼峰 | `Income` `SalaryIncome` |
| 方法名 | camelCase 小驼峰 | `getName()` `getTax()` |
| 变量名 | camelCase 小驼峰 | `income` `total` |
| 常量 | UPPER_SNAKE_CASE | `TAX_RATE` `MAX_VALUE` |
| 包名 | 全小写 + 点分隔 | `com.fuhao.extend` |

额外准则：
- 名字要有意义：`income` > `x`

### 3. 关键字全览（50+）

```
类与对象: class interface extends implements new this super abstract enum record
访问控制: public protected private
静态不可变: static final
返回值: void return
条件: if else switch case default
循环: for while do break continue
异常: try catch finally throw throws
包: package import
实例类型: instanceof
保留无功能: const goto
其他: assert strictfp native transient volatile synchronized
模块: module requires exports opens to uses provides with
```

### 4. 三个特殊字面量

`true` `false` `null` —— 不能当标识符。

### 5. var —— 上下文关键字

Java 10 引入，只在局部变量声明的特定位置是关键字。其他地方可以用它当变量名（但不推荐）。

## Code Example

```java
package com.fuhao.example;        // package: 关键字
import java.util.List;            // import: 关键字

public class Calculator {         // public, class: 关键字  Calculator: 标识符(PascalCase)
    private static final double TAX_RATE = 0.1;  // TAX_RATE: 标识符(UPPER_SNAKE)

    public static void main(String[] args) {
        double income = 5000.0;       // income: 标识符(camelCase)
        double tax = calculateTax(income);
        System.out.println("Tax: " + tax);
    }

    public static double calculateTax(double income) {
        return income * TAX_RATE;     // return: 关键字
    }
}
```

## Common Misconceptions

1. 「数字开头编译器能区分」→ 不能，`2value` 对解析器是歧义
2. 「const 和 goto 有功能」→ 只是保留字，Java 里无功能
3. 「names_like_this 也合法所以没问题」→ 会当成 Python 老手写给 Java 看
4. 「类名首字母小写不报错」→ 编译不报错，但浪费所有读代码的人的时间
5. 「关键字必须全背下来」→ 不用，IDE 自动高亮，写多了自然内化

## Socratic Check

1. 哪些合法？`myVariable` `2ndPlace` `_count` `$price` `my-name` `class` → 前三个合法（但不推荐 `_` 和 `$`），后三个非法。
2. `String String = "hello";` 能编译吗？→ 能，大小写敏感，但别这么做。
3. IDE 里 `int` 是紫色的说明什么？→ IDE 内置关键字列表帮你区分，不用硬背。

## Quick Summary

- 标识符三规则：字母数字_$，不以数字开头，不是关键字
- 命名规范：类 PascalCase、方法/变量 camelCase、常量 UPPER_SNAKE、包全小写
- Java ~50 个关键字 + true/false/null 三个特殊字面量
- 命名规范为「人」服务，不要为「编译器」起名字
