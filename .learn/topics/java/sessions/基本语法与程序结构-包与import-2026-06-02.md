## Positioning

包是 Java 程序结构的**横向维度**。「基本语法与程序结构」讲的是一个文件内部的层次（源文件 → 类 → 方法 → 语句），而**包解决的是「多个文件之间如何组织、如何找到彼此」的问题**。

位置：语言基础 → 基本语法与程序结构 → 包与 import。

---

## Analogy

把 Java 程序想像成**大型图书馆**：

- 没有包的代码 = 所有书堆在地上，每本书还不能重名
- 有包的代码 = 分类编号系统（计算机 → Java → 继承 → Student）
- 包让你**起名字不用瞻前顾后**，因为包路径已经做了区分

## Core Mechanism

### 1. package 声明

```java
package com.fuhao.extend;     // 必须是文件第一行非注释代码

public class Student extends Person {
    // ...
}
```

三条硬规则：
- package 声明必须是文件的第一行代码（注释不算）
- 每个 `.java` 文件只能有一个 package 声明
- 包路径必须和目录结构完全一致：`package com.fuhao.extend` ↔ `com/fuhao/extend/Student.java`

### 2. 包命名约定 —— 反向域名

```
com.fuhao.extend         → fuhao.com 项目
org.apache.commons.io    → apache.org 项目
```

用 `com.fuhao.xxx` 是标准做法，保证类名不会和别人冲突。

### 3. import —— 使用其他包的类

```java
package com.fuhao;

import java.util.Arrays;     // 引入 java.util 包下的 Arrays 类
import java.util.List;       // 引入 java.util 包下的 List 接口
```

import 的本质：**告诉编译器「Arrays 指的是 java.util.Arrays」**，帮你省打字。编译后 import 消失，所有类名变成全限定名。

### 4. 同包不需要 import

同一个包里的所有类，彼此默认可见，不需要 import。这就是为什么 `com.fuhao.extend.Student` 可以直接用 `Person` 而不用 import——它们都在 `com.fuhao.extend` 包下。

### 5. import 的三种写法

| 写法 | 示例 | 说明 |
|------|------|------|
| 单类导入（推荐） | `import java.util.List;` | 明确、可读 |
| 按需导入 | `import java.util.*;` | 省事，但不递归到子包 |
| 静态导入 | `import static java.lang.Math.PI;` | 可以直接写 `PI` |

`import java.util.*` 只导入 `java.util` 包下的类，**不导入子包**（如 `java.util.regex`）。

### 6. java.lang 自动导入

`System`、`String`、`Math` 不需要 import，因为 `java.lang` 包被 Java 编译器自动导入。

### 7. 默认包（无名包）

没有 package 声明的文件属于默认包。**真实项目绝不要用**：
- 其他包无法 import 默认包中的类
- 容易和第三方库冲突
- 只能用于学习 Hello World

## Code Example

跨包调用——`com.fuhao.Main` 使用 `com.fuhao.extend.Student`：

```java
package com.fuhao;

import com.fuhao.extend.Student;    // 跨包使用，必须 import

public class Main {
    public static void main(String[] args) {
        Student s = new Student();
        s.setName("小明");
        s.setAge(13);
        s.setScore(98);
        System.out.println(s.getName());
    }
}
```

编译运行：
```bash
javac com/fuhao/Main.java       # javac 自动找到依赖
java com.fuhao.Main              # 全限定类名，用 . 分隔
```

## Common Misconceptions

1. **「import 像 C 的 #include，把代码复制进来」** → 错。import 只是编译期的别名映射，不做代码复制。
2. **「import java.util.* 会拖慢程序」** → 不会。import 只在编译期起作用，`*` 导入和单个导入编译结果一样。
3. **「包名可以随便起，目录随便放」** → 不可以。package 和目录强绑定。
4. **「Java 里类不能重名」** → 应该说同一包内不能重名，不同包可以。
5. **「String 不需要 import 是编译器特殊对待」** → 是 `java.lang` 被自动 import，不是只特殊对待 String。

## Socratic Check

1. `Student.java` 用到了 `Person` 但没有 import，为什么能编译？→ 同包内类互相可见。
2. `package com.fuhao.extend` 但文件放在 `com/fuhao/other/` 下？→ 编译错误，路径不匹配。
3. `import com.fuhao.*;` 能引入 `com.fuhao.extend.Student` 吗？→ 不能，* 导入不递归到子包。

## Quick Summary

- package 声明 = 类所属的命名空间，必须与目录结构一致
- import = 引入其他包的类，本质是给编译器提供别名映射
- 同包类不需要 import，`java.lang` 自动 import
- 包命名用反向域名，保证全局唯一性
- `*` 导入不递归，不拖慢运行时
- 默认包只在学习时用
